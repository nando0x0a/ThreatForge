# Threat Hunting Pack — CVE-2026-10037
**OpenJDK .jar MIME Handler Sandbox Escape (Ubuntu / xdg-desktop-portal-gtk)**
CVSS: 8.8 HIGH [1] | No public PoC known | Tags: [HIGH] [T1] [WIDE]

---

## Vulnerability Context

This vulnerability affects OpenJDK packages on Ubuntu where mailcap is installed [2]. The `.jar` MIME handler executes files marked as executable, allowing a sandboxed application (e.g., Firefox running in a Snap sandbox) to escape confinement by leveraging the OpenURI portal via `xdg-desktop-portal-gtk` [2]. The exploit chain involves:

1. A sandboxed process invoking `xdg-open` or a URI handler for a `.jar` file
2. `xdg-desktop-portal-gtk` relaying the open request to the host mailcap handler
3. The mailcap `.jar` handler invoking the JVM outside the sandbox with attacker-controlled content
4. Arbitrary code execution in the host context

⚠️ **No specific malicious IPs, domains, or file hashes are publicly associated with exploitation of this CVE at time of writing.** Placeholder IoCs are used where applicable. Network-based indicators are not the primary detection surface for this vulnerability — process and filesystem hunting are higher yield.

## CrowdStrike Event Search

### Query 1 — Suspicious `xdg-open` / Portal Invocation from Sandboxed Process

Targets the core escape mechanism: a snap-confined process spawning `xdg-open` or `xdg-desktop-portal` interactions that result in JVM execution.

event_simpleName IN ("ProcessRollup2", "SyntheticProcessRollup2")
| search ParentBaseFileName IN ("xdg-open", "xdg-desktop-portal-gtk", "xdg-desktop-portal")
    AND FileName IN ("java", "java17", "java21", "java25", "java-*")
| eval snap_confined=if(match(ParentImageFileName, "snap"), "YES", "NO")
| stats count by ComputerName, UserName, ParentBaseFileName, FileName,
         CommandLine, snap_confined
| where snap_confined="YES"
| sort -count

**Analyst note:** `java` child processes spawned by portal/xdg components on Ubuntu hosts are high-fidelity for this CVE. Validate `snap_confined` logic against your sensor field naming.

### Query 2 — JAR File Execution via mailcap Handler Outside Expected Paths

Targets execution of `.jar` files from user-writable or temporary locations, consistent with a delivered payload being opened through the MIME handler.

event_simpleName IN ("ProcessRollup2", "SyntheticProcessRollup2")
| search FileName IN ("java", "java17", "java21", "java25")
    AND (
        match(CommandLine, "-jar\\s+/tmp/")
        OR match(CommandLine, "-jar\\s+/home/")
        OR match(CommandLine, "-jar\\s+/run/user/")
        OR match(CommandLine, "-jar\\s+/var/tmp/")
    )
| eval suspicious_jar_path=if(
    match(CommandLine, "-jar\\s+(/tmp/|/home/|/run/user/|/var/tmp/)"),
    "YES", "NO"
  )
| stats count by ComputerName, UserName, CommandLine, SHA256HashData,
         suspicious_jar_path
| where suspicious_jar_path="YES"
| sort -count

**Analyst note:** Legitimate Java apps may also launch with `-jar` from home directories. Cross-reference with `ParentBaseFileName` to prioritise portal-spawned instances. Collect `SHA256HashData` for the invoked `.jar` and submit for threat intel review.
