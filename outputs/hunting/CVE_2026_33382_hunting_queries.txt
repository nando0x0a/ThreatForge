# Threat Hunting Queries — CVE-2026-33382
**Grafana Denial of Service via Oversized Request Bodies**
*DRAFT — For analyst review prior to deployment*

---

> **Analyst Note:** CVE-2026-33382 is a 3-day-old [1] unauthenticated denial-of-service vulnerability in Grafana [2]. Exploitation involves sending oversized HTTP request bodies to affected API endpoints (attributed to missing size limits in `web.Bind` [2]), which exhausts server memory. CVSS score is 7.5 HIGH [1]. No public PoC is currently known. No CISA KEV entry is present in the provided context. The RCE flag in the priority metadata reflects network exploitability characteristics (AV:N/PR:N/UI:N) — this is a **DoS** vulnerability; analysts should treat "RCE" tagging as a metadata artefact and not infer arbitrary code execution capability without further evidence.
>
> **No specific IoCs (malicious IPs, domains, or file hashes) are available for this CVE.** Queries below use behavioural and volumetric patterns. Placeholder values are used where specific IoCs are unavailable.

## CrowdStrike Event Search

### Query 1 — Anomalous Inbound HTTP Connections to Grafana Process Port

event_simpleName=NetworkConnectIP4
| search LocalPort IN ("3000", "8080", "443") AND (ProcessName="grafana-server" OR ProcessName="grafana")
| bucket _time span=5m
| stats count as ConnectionCount by _time, ComputerName, RemoteIP, LocalPort
| where ConnectionCount > 100
| sort -ConnectionCount
*Hunts for high-frequency inbound connections to Grafana's default port (3000) or common proxy ports. Sustained flooding from a single RemoteIP is a key indicator of large-payload DoS attempts [2].*

---

### Query 2 — Grafana Process Memory or Crash Signals (OOM / Service Restart)

event_simpleName IN ("ProcessRollup2", "ProcessSelfDeleted")
| search ImageFileName CONTAINS "grafana"
| eval RestartFlag=if(event_simpleName="ProcessSelfDeleted", "TERMINATED", "SPAWNED")
| stats count by ComputerName, ImageFileName, RestartFlag, UserName
| where count > 3
| sort -count
*Repeated grafana-server process terminations and respawns may indicate the process is crashing due to memory exhaustion caused by oversized request body ingestion [2]. More than 3 cycles in a short window should be investigated.*

---

### Query 3 — Unusual Child Processes Spawned from Grafana (Post-Exploitation Baseline)

event_simpleName=ProcessRollup2
| search ParentBaseFileName IN ("grafana-server", "grafana")
| stats values(FileName) as ChildProcesses, count by ComputerName, ParentBaseFileName, UserName
| where ChildProcesses NOT IN ("grafana-server", "grafana-cli")
| sort -count
*No RCE has been confirmed for this CVE [2], but this query establishes a baseline for unexpected child processes from the Grafana parent — useful if the vulnerability's impact is later revised or chained with another issue. Flag any shell, script interpreter, or network utility spawned from Grafana.*

## nfdump Netflow Queries

### Query 1 — High-Volume Traffic to Grafana Default Port (DoS Flood Detection)

nfdump -r /var/log/netflow/nfcapd.current \
  -f 'proto tcp and dst port 3000' \
  -A srcip,dstip,dstport \
  -s record/bytes -n 30 \
  -o "fmt:%ts %sa %da %dp %byt %pkt"
*Identifies the top 30 source IPs sending the highest byte volumes to Grafana's default port 3000. A single source IP responsible for an outsized proportion of bytes — especially with few packets (large payload per request) — is consistent with oversized body injection [2].*

---

### Query 2 — Abnormal Bytes-per-Flow Ratio on Grafana API Port (Large Payload Signature)

nfdump -r /var/log/netflow/nfcapd.current \
  -f 'proto tcp and dst port 3000 and bytes > 5000000' \
  -A srcip,dstip \
  -s record/bytes -n 20 \
  -o "fmt:%ts %sa %da %byt %pkt %fl"
*Filters for individual flows exceeding 5 MB directed at port 3000. Legitimate Grafana API calls are typically small; flows in the megabyte range per connection are anomalous and may represent the large-body payloads described in the advisory [2]. Adjust the byte threshold to match your environment's baseline.*

---

### Query 3 — Rapid Connection Rate from Single Source to Grafana (Pre-Exhaustion Pattern)

nfdump -r /var/log/netflow/nfcapd.current \
  -f 'proto tcp and dst port 3000 and flags S' \
  -A srcip \
  -s flows/flows -n 20 \
  -o "fmt:%ts %sa %fl %byt" \
  -t 2026-01-01/2026-12-31
*Counts TCP SYN flows per source IP to port 3000. A high SYN flow count from a single external IP may indicate scripted, repeated large-body POST flooding. Adjust the `-t` time window to the incident period. Replace port 3000 with your deployment's actual Grafana listener port if different.*

## Hunting Notes

### What to Look For

| Signal | Relevance |
|---|---|
| High inbound byte volumes to port 3000/8080 from a single external IP | Core exploit behaviour — large request bodies to API endpoints [2] |
| Grafana process repeated crash/restart cycles | Memory exhaustion DoS symptom [2] |
| Single-source IPs sustaining >100 connections per 5-minute window | Scripted flooding pattern |
| Flows with very high bytes-per-packet ratio (large payloads, few packets) | Distinguishes large-body DoS from connection-flood DoS |
| Unexpected child processes from grafana-server | Precautionary; no confirmed RCE at this time [2] |

### False Positive Considerations

- **Grafana bulk data ingestion:** Legitimate Prometheus remote-write, Loki log push, or large dashboard exports may produce high-byte flows to port 3000. Correlate with scheduled job timing and known data pipeline source IPs before escalating.
- **Load balancer health checks:** High connection counts from internal monitoring systems or load balancers are expected. Exclude known health-check IPs from connection-rate queries.
- **Grafana upgrades/restarts:** Planned maintenance will produce process termination/respawn events. Cross-reference with change management records.
- **Port variation:** Grafana may run behind a reverse proxy on 443 or 8080. Validate the actual listener port in your environment and update queries accordingly.

### Escalation Criteria

1. **Immediate escalation:** Grafana service confirmed unavailable or degraded performance correlated with anomalous inbound traffic volumes.
2. **Escalate for investigation:** Any single external IP responsible for >50% of byte volume to port 3000 within a 15-minute window.
3. **Escalate for investigation:** Three or more grafana-server process restarts within 30 minutes without a corresponding change record.
4. **Precautionary escalation:** Any child process (shell, curl, wget, python, etc.) with Grafana as parent — even though RCE is not confirmed for this CVE [2], this would warrant immediate triage.
5. **Patch priority:** This vulnerability is 3 days old [1] with no public PoC currently known; however, the unauthenticated, network-exploitable nature (PR:N, UI:N) [1] means patch urgency is high. Confirm Grafana version and apply vendor patch per [2].
