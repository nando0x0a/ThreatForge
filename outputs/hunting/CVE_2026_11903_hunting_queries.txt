## Severity & Context Note

CVE-2026-11903 is a stored XSS vulnerability in the Progress MOVEit Transfer Ad Hoc module, scored **8.0 (HIGH)**[1]. Affected versions span three release trains: 2026.0.0 before 2026.0.1, 2025.1.0 before 2025.1.4, and 2025.0.0 before 2025.0.8[2]. The vulnerability arises from improper neutralisation of input during web page generation[3]. Exploitation requires user interaction (a victim must load the poisoned page), but stored XSS in a managed file transfer platform represents a meaningful risk — injected scripts execute in the browser context of authenticated MOVEit users, enabling session token theft, credential harvesting, or redirection to secondary payloads. No public PoC is currently known, and the vulnerability is 6 days old — early-stage hunting is appropriate. Priority Tier is **STANDARD** with tags **[HIGH] [T1]**.

> **No confirmed malicious IPs, domains, or file hashes are available for this CVE at time of writing.** Placeholder values are used throughout. Substitute confirmed IoCs as threat intelligence matures.

## CrowdStrike Event Search

### Query 1 — Outbound Connections from MOVEit Host to Suspicious Destinations
Targets potential script-injected callbacks (e.g., cookie exfiltration beaconing) originating from the MOVEit server process or from browsers of users who loaded a poisoned Ad Hoc page.

event_simpleName=NetworkConnectIP4
| search (LocalAddressIP4 IN ("MOVEIT-SERVER-IP-1","MOVEIT-SERVER-IP-2")
    OR ComputerName IN ("MOVEIT-HOSTNAME-1","MOVEIT-HOSTNAME-2"))
  RemotePort IN (80, 443, 8080, 8443)
  RemoteAddressIP4 != "10.0.0.0/8"
  RemoteAddressIP4 != "172.16.0.0/12"
  RemoteAddressIP4 != "192.168.0.0/16"
| eval suspicion=if(RemoteAddressIP4 IN ("x.x.x.x","x.x.x.y"), "KNOWN_BAD_IP",
    if(match(RemotePort,"(8080|8443)"), "UNUSUAL_PORT", "REVIEW"))
| stats count min(_time) as firstSeen max(_time) as lastSeen
    by ComputerName, UserName, RemoteAddressIP4, RemotePort, suspicion
| sort -count
> **Analyst note:** Replace `MOVEIT-SERVER-IP-*` and `MOVEIT-HOSTNAME-*` with your actual MOVEit Transfer server assets. RFC1918 exclusions reduce noise from internal traffic; remove if your MOVEit instance communicates intra-network to user endpoints.

### Query 2 — Browser/Script Process Spawning Unusual Children on MOVEit User Endpoints
Stored XSS executing in a victim's browser may trigger download-and-execute chains. This query looks for browser processes spawning scripting engines or command shells on endpoints belonging to users who accessed MOVEit recently.

event_simpleName=ProcessRollup2
  ParentBaseFileName IN ("chrome.exe","msedge.exe","firefox.exe","iexplore.exe",
                         "opera.exe","brave.exe")
  FileName IN ("powershell.exe","pwsh.exe","cmd.exe","wscript.exe","cscript.exe",
               "mshta.exe","rundll32.exe","regsvr32.exe","certutil.exe","curl.exe",
               "wget.exe","bitsadmin.exe")
| where isnotnull(UserName)
| eval riskFlag=if(FileName IN ("mshta.exe","wscript.exe","cscript.exe",
                                "regsvr32.exe","certutil.exe"), "HIGH_RISK",
              if(FileName IN ("powershell.exe","pwsh.exe","cmd.exe"), "MEDIUM_RISK",
              "LOW_RISK"))
| stats count values(CommandLine) as CommandLines min(_time) as firstSeen
    by ComputerName, UserName, ParentBaseFileName, FileName, riskFlag
| where count > 0
| sort riskFlag -count
> **Analyst note:** Correlate `UserName` values against MOVEit access logs for the period since CVE disclosure (approximately 6 days). Users with MOVEit sessions who show browser-spawned scripting activity are highest priority for review.

### Query 3 — Suspicious JavaScript/Credential Harvest File Drops in Browser Profile Directories
Injected XSS may write harvested tokens or stage payloads in browser-accessible paths. This query targets file creation events in browser data directories or temp paths from browser processes on known MOVEit user endpoints.

event_simpleName=AsepValueChange OR event_simpleName=NewExecutableWritten
  OR event_simpleName=SuspiciousFile
| search (TargetDirectoryName MATCHES "(?i)(\\AppData\\Roaming\\|\\AppData\\Local\\Temp\\|\\Downloads\\)")
  ImageFileName MATCHES "(?i)(chrome|msedge|firefox|iexplore)"
| eval ext=lower(mvindex(split(TargetFileName,"."), -1))
| where ext IN ("exe","dll","ps1","vbs","js","bat","hta","wsf","scr","lnk")
| stats count values(TargetFileName) as Files values(TargetDirectoryName) as Dirs
    min(_time) as firstSeen
    by ComputerName, UserName, ImageFileName
| sort -count
> **Analyst note:** `.js` extension hits will have high false-positive rates in browser caches. Focus initial triage on `.exe`, `.ps1`, `.hta`, `.dll` drops. Adjust `TargetDirectoryName` regex to match your environment's Windows profile path conventions.

### Query 1 — Outbound Traffic from MOVEit Server to Known-Bad Destinations
Targets exfiltration callbacks or C2 check-ins from the MOVEit Transfer server itself.

nfdump -r /var/log/netflow/nfcapd.current \
  -t 2026-06-01/2026-06-30 \
  -f 'proto tcp \
      and src ip MOVEIT.SERVER.IP.HERE \
      and (dst ip x.x.x.x or dst ip x.x.x.y) \
      and (dst port 80 or dst port 443 or dst port 8080 or dst port 8443)' \
  -o "fmt:%ts %te %sa %da %sp %dp %pkt %byt %fl" \
  -s record/bytes -n 50
