## CrowdStrike Event Search

### Query 1 — Suspicious Child Processes Spawned by Apache HTTP Server (RCE Detection)
*Targets post-exploitation command execution via CGI or mod_cgi abuse*

event_simpleName IN ("ProcessRollup2", "SyntheticProcessRollup2")
| search ParentBaseFileName IN ("httpd", "apache2", "apache", "httpd.worker")
| search FileName IN ("sh", "bash", "dash", "python", "python3", "perl",
                      "curl", "wget", "nc", "ncat", "netcat", "id", "whoami",
                      "uname", "passwd", "env", "awk", "php")
| eval SuspiciousCmdLine=if(
    match(CommandLine, "(?i)(curl|wget|bash|/tmp/|/dev/shm|chmod|base64|python|perl)"),
    "HIGH", "MEDIUM")
| stats count by ComputerName, ParentBaseFileName, FileName,
         CommandLine, UserName, SuspiciousCmdLine
| sort -count
> **Analyst Note:** Apache spawning interactive shells or download utilities (curl/wget) is a strong
> indicator of CVE-2021-42013 exploitation. Validate against known CGI script inventory before escalating.

### Query 2 — File System Artefacts — Web Shell or Payload Drops in Apache-Accessible Paths
*Targets file creation events in typical Apache document roots and CGI directories*

event_simpleName IN ("NewExecutableWritten", "PeFileWritten", "NewScriptWritten")
| search TargetDirectoryName IN (
    "/var/www/", "/var/www/html/", "/usr/lib/cgi-bin/",
    "/srv/www/", "/opt/apache/", "/usr/local/apache2/htdocs/",
    "/usr/local/apache2/cgi-bin/")
| eval FileExtension=lower(mvindex(split(TargetFileName, "."), -1))
| eval SuspiciousExt=if(
    FileExtension IN ("php", "pl", "py", "sh", "cgi", "rb", "jsp", "elf"),
    "YES", "NO")
| where SuspiciousExt="YES"
| stats count by ComputerName, UserName, TargetDirectoryName,
         TargetFileName, SHA256HashData
| sort -count
> **Analyst Note:** Script or executable drops into CGI-bin or document root by the Apache process user
> (typically `www-data`, `apache`, `nobody`) warrant immediate triage. Cross-reference SHA256 against threat intel.

### Query 3 — Outbound Network Connections from Apache Process (Reverse Shell / C2 Beacon)
*Targets unexpected egress initiated by the Apache server process — indicative of reverse shell or C2*

event_simpleName="NetworkConnectIP4"
| search LocalAddressIP4=*
| search ImageFileName IN (
    "/usr/sbin/apache2", "/usr/sbin/httpd",
    "/usr/local/apache2/bin/httpd",
    "/opt/apache2/bin/httpd")
| where RemotePort NOT IN (80, 443, 8080, 8443)
  OR (RemotePort IN (443, 80) AND isnotnull(RemoteAddressIP4)
      AND NOT cidrmatch("10.0.0.0/8", RemoteAddressIP4)
      AND NOT cidrmatch("172.16.0.0/12", RemoteAddressIP4)
      AND NOT cidrmatch("192.168.0.0/16", RemoteAddressIP4))
| stats count dc(RemoteAddressIP4) AS UniqueRemoteIPs
        by ComputerName, ImageFileName, RemoteAddressIP4, RemotePort, UserName
| where UniqueRemoteIPs > 1 OR count > 5
| sort -count
> **Analyst Note:** Apache initiating outbound connections to non-standard ports or unexpected external IPs
> is a high-fidelity indicator. Baseline legitimate update/OCSP traffic before suppressing results.

## nfdump Netflow Queries

### Query 1 — Inbound HTTP/HTTPS Traffic to Apache Servers with Exploit-Sized Requests
*Targets traffic patterns consistent with path traversal payloads (encoded `..%2f` sequences tend to inflate URI length)*

nfdump -r /var/log/netflow/nfcapd.current \
  -f 'proto tcp and dst port in [80,443,8080,8443] and dst ip x.x.x.x/32' \
  -A srcip,dstip,dstport \
  -s record/packets \
  -o "fmt:%ts %sa %da %dp %pkt %byt %bps" \
  -n 50
> **Analyst Note:** Replace `x.x.x.x/32` with your Apache server IP(s) or CIDR range.
> Flag source IPs generating >50 requests in short windows — consistent with automated scanning/exploitation.
> Specific exploit URIs (`/cgi-bin/.%2e/.%2e/bin/sh`) are not visible in netflow; use WAF/proxy logs to correlate.

### Query 2 — Anomalous Outbound Connections from Apache Server (Reverse Shell Detection)
*Targets unexpected egress from web server hosts — reverse shell activity generates low-volume, persistent flows*

nfdump -r /var/log/netflow/nfcapd.current \
  -f 'proto tcp and src ip x.x.x.x/32 and not dst port in [80,443,25,53,123,8080,8443]' \
  -A srcip,dstip,dstport \
  -s record/flows \
  -o "fmt:%ts %sa %da %dp %pkt %byt %fl %td" \
  -n 30
> **Analyst Note:** Replace `x.x.x.x/32` with your Apache server IP(s). Outbound flows from web servers
> on ports such as 4444, 1337, 9001, 31337, or any ephemeral non-standard port are high-priority leads.
> Long-duration, low-packet flows are characteristic of interactive reverse shells.

### Query 3 — Scanning / Reconnaissance Traffic Preceding Exploitation Attempts
*Targets high-rate inbound connection attempts consistent with automated vulnerability scanning for CVE-2021-42013*

nfdump -r /var/log/netflow/nfcapd.current \
  -f 'proto tcp and dst port in [80,443,8080,8443] and dst ip x.x.x.x/32 and flags S' \
  -A srcip \
  -s record/flows \
  -o "fmt:%ts %sa %fl %pkt %byt" \
  -n 25 | awk '$3 > 20 {print}'
> **Analyst Note:** Replace `x.x.x.x/32` with your Apache server IP(s). Source IPs with >20 SYN flows
> in a single capture window suggest automated scanning. Cross-reference against threat intel feeds
> (GreyNoise, Shodan) for known scanner IPs targeting Apache 2.4.49/2.4.50.
