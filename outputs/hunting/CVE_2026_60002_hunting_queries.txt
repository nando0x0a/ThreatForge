# Threat Hunting Queries — CVE-2026-60002

**Vulnerability Summary:** OpenSSH versions prior to 10.4 contain a use-after-free triggered when a server changes its host key during SSH key re-exchange (rekey). This can result in client-side memory corruption. The vulnerability is network-exploitable with no authentication and no user interaction required [1][4]. CVSS score is 7.7 HIGH [1]. Fixed in OpenSSH 10.4 / 10.4p1 released 2026-07-06 [2][3].

> ⚠️ **No confirmed public IoCs (IPs, domains, file hashes) are available at time of writing — this CVE is 5 days old.** All network-based IoC fields below use placeholders. Replace with threat-intel-confirmed values before running. Behavioural and process-pattern queries remain valid as written.

## CrowdStrike Event Search

### Query 1 — Outbound SSH Connections to Known Malicious IPs (Network IoC Pivot)

> Targets client-side SSH connections to attacker-controlled servers that may serve as malicious SSH hosts capable of triggering the rekey host-key-change condition.

event_simpleName=NetworkConnectIP4
| where RemotePort IN (22, 2222)
| where RemoteIP IN ("x.x.x.x", "y.y.y.y")
| stats count, values(LocalPort) AS LocalPorts,
         dc(ComputerName) AS UniqueHosts
         by ComputerName, UserName, RemoteIP, RemotePort, LocalAddressIP4
| where count > 1
| sort -count

**Analyst notes:** Substitute `x.x.x.x`, `y.y.y.y` with confirmed malicious IPs from threat intel. Port 2222 included as a common non-standard SSH port used in attacker infrastructure.

### Query 2 — Post-Exploitation Process Execution Patterns Following SSH Client Activity

> Targets anomalous child process spawning from the SSH client binary (`ssh`) that may indicate successful memory corruption and code execution on the client host. This is the highest-value behavioural signal for this CVE given the client-side RCE vector [1][4].

event_simpleName=ProcessRollup2
| where ParentBaseFileName IN ("ssh", "ssh-agent", "scp", "sftp")
| where FileName NOT IN ("ssh", "scp", "sftp", "ssh-keygen",
                          "ssh-add", "ssh-agent", "bash", "sh",
                          "known_hosts")
| eval suspicion=case(
    FileName IN ("curl", "wget", "python3", "python", "perl", "ruby"),
    "HIGH — download/interpreter spawned from SSH client",
    FileName IN ("nc", "ncat", "nmap", "socat"),
    "HIGH — network tool spawned from SSH client",
    FileName IN ("chmod", "chown", "id", "whoami", "uname"),
    "MEDIUM — recon/permission tool spawned from SSH client",
    true(), "LOW — review"
  )
| stats count, values(CommandLine) AS Commands,
         values(suspicion) AS SuspicionLevel
         by ComputerName, UserName, ParentBaseFileName,
            FileName, MD5HashData
| where suspicion != "LOW — review"
| sort -count

**Analyst notes:** False positives expected where legitimate automation scripts invoke SSH then pipe to shell utilities. Correlate `CommandLine` values and `UserName` against expected automation accounts. Escalate on any interpreter or download-tool spawn with an interactive or service `UserName`.

### Query 3 — SSH Client Process Writing Unexpected Files (Post-Exploitation Artefact Detection)

> Targets file write activity originating from the SSH client process, which would be anomalous and may indicate payload drop following memory corruption exploitation.

event_simpleName=PeFileWritten OR event_simpleName=GenericFileWritten
| lookup local=true aid_master.csv aid OUTPUT ComputerName, OS
| where OS IN ("Linux", "macOS")
| join type=inner ComputerName
    [search event_simpleName=ProcessRollup2
     ParentBaseFileName IN ("ssh", "sftp", "scp")
     | fields ComputerName, TargetProcessId, FileName AS SSHBinary]
| where NOT match(TargetFileName, "^(/home/[^/]+/\.ssh/|/root/\.ssh/|/tmp/ssh-)")
| stats count, values(TargetFileName) AS WrittenFiles,
         values(SSHBinary) AS ParentSSHProcess
         by ComputerName, UserName, TargetFileName
| sort -count

**Analyst notes:** Legitimate SSH writes go to `~/.ssh/known_hosts` and temp socket files. Any write to `/tmp/`, `/var/tmp/`, `/dev/shm/`, or executable paths (`/usr/local/bin/`, `/usr/bin/`) is high-suspicion. Escalate immediately on writes to cron directories or systemd unit paths.

## nfdump Netflow Queries

### Query 1 — Outbound SSH Traffic to Known Malicious IPs

> Identifies flows from internal hosts to attacker-controlled SSH servers that may be delivering the malicious rekey trigger.

nfdump -r /var/log/netflow/nfcapd.current \
  -f 'proto tcp and dst port 22 and \
      (dst ip x.x.x.x or dst ip y.y.y.y)' \
  -o 'fmt:%ts %te %sa %da %sp %dp %pkt %byt %fl' \
  -s record/bytes -n 50

**Analyst notes:** Replace `x.x.x.x`, `y.y.y.y` with confirmed threat-intel IPs. Also run against port 2222 for non-standard SSH. A single TCP session establishing to port 22 on a suspicious IP with an unusually short duration (< 5 seconds) and low byte count may indicate a failed exploit or probe; longer sessions with asymmetric byte counts warrant deeper inspection.

---

### Query 2 — Anomalous SSH Rekey Traffic Volume / Session Duration Anomalies

> The exploit requires a rekey event; sessions exhibiting unusually high rekey frequency or unusual byte-count patterns relative to session duration may be indicative. This query surfaces high-packet-count, asymmetric SSH flows for review.

nfdump -r /var/log/netflow/nfcapd.current \
  -f 'proto tcp and dst port 22 and duration > 5 and bytes > 50000' \
  -A srcip,dstip,dstport \
  -o 'fmt:%sa %da %dp %pkt %byt %td %fl' \
  -s record/bytes -n 30 \
  -t 2026-07-01/2026-07-12

**Analyst notes:** Legitimate SSH sessions to internal servers will appear here — baseline against known admin jump-hosts and CI/CD pipeline source IPs. Flag any flow from a workstation (non-server) source IP to an **external** destination on port 22 with duration > 60 seconds and asymmetric byte ratio (upstream >> downstream, which is atypical for interactive SSH). Rekey events in SSH generate small symmetric handshake bursts; anomalous rekey-heavy sessions may show as elevated packet counts relative to transferred bytes.
