# Threat Hunting Queries — CVE-2026-59221
**Open WebUI Path Traversal via Percent-Encoded Proxy Paths**
**Severity:** 7.7 HIGH [1] | **Age:** 4 days | **PoC:** Public [5] | **Tier:** MONITOR

---

## Background

CVE-2026-59221 affects Open WebUI versions 0.9.6 through 0.10.0 [1][4]. The vulnerability resides in `_sanitize_proxy_path`, which insufficiently decodes percent-encoded sequences (e.g., `%2F`, `%2E`, double-encoding variants like `%252F`) before path validation, enabling path traversal to arbitrary server paths [1][3]. A public proof-of-concept exists [5], and a patch has been committed [2]. Open WebUI typically runs on HTTP ports 3000 or 8080. No specific malicious IPs or domains are confirmed at this time — placeholder values are used below.

## CrowdStrike Event Search

### Query 1 — HTTP Requests with Percent-Encoded Traversal Patterns in URL (Web Process Behaviour)

Targets processes spawned by Open WebUI (Python/Uvicorn) that may indicate post-exploitation command execution triggered via path traversal.

event_simpleName IN ("ProcessRollup2", "SyntheticProcessRollup2")
| search ParentBaseFileName IN ("python3", "python", "uvicorn", "gunicorn")
| search CommandLine IN ("*cat*", "*wget*", "*curl*", "*chmod*", "*bash*", "*/etc/passwd*", "*/etc/shadow*", "*id*", "*whoami*")
| table _time, ComputerName, UserName, ParentBaseFileName, FileName, CommandLine, SHA256HashData
| sort -_time

> **Rationale:** Path traversal in a web proxy layer may be chained with server-side file reads or, in misconfigured environments, RCE. This targets child processes of the Python runtime hosting Open WebUI [1][3].

### Query 2 — Network Connections from Open WebUI Host Process to Unexpected External Destinations

Targets outbound connections from the Open WebUI Python process, which may indicate exfiltration or C2 callback following exploitation.

event_simpleName="NetworkConnectIP4"
| search LocalAddressIP4!=127.0.0.1
| join aid [search event_simpleName IN ("ProcessRollup2","SyntheticProcessRollup2")
    ParentBaseFileName IN ("python3","python","uvicorn","gunicorn")
    | table aid, ContextProcessId, FileName]
| where RemotePort IN (80, 443, 4444, 8080, 9001)
| stats count by ComputerName, UserName, FileName, RemoteAddressIP4, RemotePort
| sort -count

> **Rationale:** Successful path traversal may facilitate credential theft (e.g., `/etc/passwd`, `.env` files) followed by exfiltration or attacker callback [1][5]. No confirmed malicious IPs are available; analyst should overlay current threat intel feeds.

## nfdump Netflow Queries

### Query 1 — Inbound Traffic to Open WebUI Default Ports with High Request Rates (Scanning/Exploit Attempts)

nfdump -r /var/log/netflow/nfcapd.current \
  -f 'proto tcp and dst port in { 3000 8080 } and flags S' \
  -s srcip/flows -n 50 \
  -o "fmt:%ts %sa %da %dp %pkt %byt %fl"

> **Rationale:** Active exploitation of a public PoC [5] would likely manifest as elevated SYN-flagged flows to Open WebUI's default listening ports (3000, 8080) from novel source IPs. High flow counts from single sources warrant investigation.

### Query 2 — Outbound Data Transfer Anomalies from Open WebUI Hosts (Potential Exfiltration)

nfdump -r /var/log/netflow/nfcapd.current \
  -f 'proto tcp and src ip x.x.x.x/24 and src port in { 3000 8080 } and bytes > 500000' \
  -s dstip/bytes -n 20 \

> **Rationale:** Successful path traversal against Open WebUI [1] could result in large file reads (configuration files, model weights, credentials). Anomalously large outbound byte counts from service ports to external IPs indicate potential data exfiltration. Replace `x.x.x.x/24` with your Open WebUI server subnet.

### Query 3 — Repeated Short-Duration Flows Consistent with Traversal Enumeration

nfdump -r /var/log/netflow/nfcapd.current \
  -f 'proto tcp and dst port in { 3000 8080 } and duration < 2 and bytes > 0' \
  -a -A srcip,dstip \
  -s srcip/flows -n 30 \
  -o "fmt:%ts %sa %da %dp %fl %byt"

> **Rationale:** Path traversal exploitation via crafted proxy paths [1][3] often involves iterative enumeration attempts (e.g., `../`, `%2F..%2F`, double-encoding variants). This manifests as many short-lived TCP flows from the same source, distinct from normal browsing behaviour.

## Hunting Notes

### What to Look For

| Signal | Details |
|---|---|
| URL patterns | Requests containing `%2e%2e`, `%252f`, `%252e`, `..%2f`, `%2f..` in proxy path parameters targeting the Open WebUI API |
| Process lineage | Python/Uvicorn spawning shells, `cat`, `curl`, `wget` as child processes |
| File access | Reads to `/etc/passwd`, `/etc/shadow`, `.env`, application secrets outside the webroot |
| Network spikes | Elevated inbound flows to ports 3000/8080; outbound spikes to unknown IPs |
| Auth context | Requests arriving with or without valid session tokens — the vulnerability description does not restrict exploitation to authenticated users [1][4] |

### False Positive Considerations

- **Legitimate URL encoding:** Some API clients and proxies encode path characters legitimately. Filter by double-encoding or unusual nesting patterns rather than single `%2F` alone.
- **Penetration testing:** Coordinate with security teams conducting authorised testing on the same ports.
- **Log aggregation tools:** Automated health-check or monitoring agents may generate frequent short-lived flows to ports 3000/8080.
- **Python processes:** In environments where Python is widely used, process lineage queries may return noise — narrow by specific parent process names tied to Open WebUI deployment.

### Escalation Criteria

1. Any confirmed read of `/etc/passwd`, `/etc/shadow`, `.env`, or application secrets from the Open WebUI process.
2. Child process spawned from Uvicorn/Gunicorn executing shell commands.
3. Outbound data transfer > 1 MB from Open WebUI service ports to external IPs
