## CrowdStrike Event Search

### Query 1 — Suspicious Child Process Spawned by Apache httpd (RCE Indicator)
Targets post-exploitation command execution via CGI/mod_cgi abuse, the primary RCE vector for this CVE.

event_simpleName IN ("ProcessRollup2", "SyntheticProcessRollup2")
  ParentBaseFileName="httpd"
| where FileName IN ("sh", "bash", "dash", "curl", "wget", "python", "python3",
                     "perl", "ruby", "nc", "ncat", "nmap", "id", "whoami",
                     "uname", "cat", "chmod", "chown")
| stats count, values(CommandLine) AS CommandLines,
        values(FileName) AS ChildProcesses,
        dc(FileName) AS UniqueChildren
    by ComputerName, UserName, ParentBaseFileName, ParentProcessId
| where count > 0
| sort -count
> **Analyst note:** Apache spawning shells or reconnaissance binaries directly is high-fidelity. Review `CommandLines` for path traversal strings (`../../`), `/etc/passwd` reads, or reverse shell patterns.

### Query 2 — Path Traversal Patterns in HTTP Request Arguments / Command Lines
Targets traversal strings observed in process command lines or arguments passed to Apache worker processes.

event_simpleName IN ("ProcessRollup2", "SyntheticProcessRollup2")
  ParentBaseFileName="httpd"
| where CommandLine MATCHES "(\\.\\./){2,}"
   OR CommandLine MATCHES "%2e%2e[%/]"
   OR CommandLine MATCHES "%252e%252e"
   OR CommandLine MATCHES "/cgi-bin/"
   OR CommandLine MATCHES "/icons/"
| stats count, values(CommandLine) AS CommandLines,
        values(FileName) AS ChildProcesses
    by ComputerName, UserName, ParentBaseFileName, timestamp
| sort -timestamp
> **Analyst note:** Double-URL-encoded traversal (`%252e%252e`) is the specific bypass used in CVE-2021-41773 and its successor CVE-2021-42013. Prioritise these matches. Note that raw HTTP request strings may not appear in process telemetry on all configurations — cross-reference with web server access logs.

### Query 3 — Outbound Network Connections Initiated by Apache Worker Processes (Reverse Shell / Beacon)
Targets callback activity following successful RCE exploitation.

event_simpleName="NetworkConnectIP4"
  LocalAddressIP4 != "127.0.0.1"
| lookup process_context on ContextProcessId
| where ParentBaseFileName="httpd" OR BaseFileName="httpd"
| where RemotePort IN ("4444", "4445", "1234", "8080", "8443", "9001",
                       "443", "80", "53")
| stats count, values(RemoteAddressIP4) AS RemoteIPs,
        values(RemotePort) AS Ports,
        dc(RemoteAddressIP4) AS UniqueRemoteIPs
    by ComputerName, UserName, BaseFileName, ParentBaseFileName
| where count > 0
| sort -UniqueRemoteIPs
> **Analyst note:** Apache worker processes initiating outbound connections is abnormal in most environments. Ports 4444 and 1234 are common reverse shell defaults; 443/80 may indicate C2 over legitimate ports. No specific malicious IPs are confirmed for this CVE — substitute `RemoteAddressIP4 IN ("x.x.x.x")` if threat intel IPs are available.

## nfdump Netflow Queries

### Query 1 — High-Rate Inbound HTTP/HTTPS Requests to Apache Servers (Exploitation Probe Detection)
Targets scanning and exploitation attempts against port 80/443 on known Apache hosts.

nfdump -r /var/log/netflow/nfcapd.current \
  -f 'proto tcp and (dst port 80 or dst port 443) and dst ip x.x.x.x/24' \
  -A srcip,dstip,dstport \
  -s record/flows -n 50 \
  -o "fmt:%ts %sa %da %dp %fl %byt %pkt"
> **Analyst note:** Replace `x.x.x.x/24` with your Apache server subnet. Exploitation of CVE-2021-41773 requires only a single crafted HTTP GET request; look for sources sending repeated requests (path fuzzing) rather than relying on volume alone. Sort by flow count per source IP to identify scanners.

### Query 2 — Outbound Connections from Apache Server Hosts to Uncommon Ports (Post-Exploitation Callback)
Targets reverse shell or beacon traffic originating from compromised Apache servers.

nfdump -r /var/log/netflow/nfcapd.current \
  -f 'proto tcp and src ip x.x.x.x and not (dst port 80 or dst port 443 or dst port 53 or dst port 25) and flags S' \
  -A srcip,dstip,dstport \
  -s record/flows -n 30 \
  -o "fmt:%ts %sa %da %dp %fl %byt %pkt"
> **Analyst note:** Replace `src ip x.x.x.x` with the specific Apache server IP(s). Web servers initiating outbound SYN connections to non-standard ports is anomalous. Flag connections to ports such as 4444, 1234, 9001, and any ephemeral ports on external IPs for investigation. Substitute `x.x.x.x` in `dst ip` with known-bad IPs from threat intel feeds if available.

### Query 3 — Traffic Volume Anomaly on CGI-Relevant Paths (Lateral/Exfiltration Indicator)
Targets abnormally large outbound byte volumes from the Apache server, indicating potential data exfiltration following RCE.

nfdump -r /var/log/netflow/nfcapd.current \
  -f 'proto tcp and src ip x.x.x.x and dst port 443 and bytes > 500000' \
  -A srcip,dstip,dstport \
  -s record/bytes -n 20 \
  -o "fmt:%ts %sa %da %dp %fl %byt %pkt" \
  -t 2024-01-01/2024-12-31
> **Analyst note:** Adjust `-t` to your investigation window. The 500,000-byte threshold is a starting point — baseline your Apache server's normal outbound data volumes and tune accordingly. Large outbound flows to new or rare external IPs following an inbound spike warrant immediate escalation.

## Hunting Notes

### What to Look For

| Indicator | Significance |
|-----------|-------------|
| `httpd` spawning `sh`, `bash`, `curl`, `wget` | Direct RCE via mod_cgi — high confidence |
| URL-encoded traversal strings (`%2e%2e`, `%252e`) in logs/cmdlines | CVE-2021-41773 / CVE-2021-42013 specific bypass pattern |
| Reads of `/etc/passwd`, `/etc/shadow`, `/proc/self/environ` | Traversal-only (non-RCE) exploitation confirming access |
| Apache worker processes with unexpected outbound TCP SYN | Reverse shell or C2 beacon post-RCE |
| Spikes in 4xx/5xx HTTP responses followed by 200 responses to `/cgi-bin/` | Exploitation attempt succeeded after probing |
| New files created in `/tmp`, `/var/tmp`, or web root by `www-data`/`apache` | Webshell or tool staging |

### False Positive Considerations

- **Legitimate CGI usage:** Environments that intentionally use CGI scripts will generate `httpd`→`sh` process chains; validate against approved CGI inventory before escalating.
