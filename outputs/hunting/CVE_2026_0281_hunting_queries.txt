## CrowdStrike Event Search

### Query 1 — Outbound Network Connections from PAN-OS Management Hosts to Suspicious Destinations
*Targets: Session tokens exfiltrated via user-clicked malicious link, resulting in outbound callouts from management workstations.*

event_simpleName=NetworkConnect
| search RemotePort IN ("443", "80", "8443", "8080")
  RemoteIP IN ("x.x.x.x")
| where isnotnull(UserName)
| eval connection_summary=ComputerName+"|"+UserName+"|"+RemoteIP+":"+RemotePort
| stats count AS ConnectionCount, earliest(timestamp) AS FirstSeen, latest(timestamp) AS LastSeen
    by ComputerName, UserName, RemoteIP, RemotePort
| where ConnectionCount < 5
| sort -LastSeen
> **Analyst note:** `x.x.x.x` is a placeholder — no confirmed malicious IPs are associated with CVE-2026-0281 at this time. Substitute confirmed IoCs when available. Low-count filter (`< 5`) targets beaconing and one-off exfiltration rather than routine traffic.

### Query 2 — Browser Process Spawning Unusual Child Processes on Management Hosts
*Targets: Post-exploitation activity following a user clicking a malicious link that delivers the session token.*

event_simpleName=ProcessRollup2
| where ParentBaseFileName IN ("chrome.exe", "firefox.exe", "msedge.exe",
    "iexplore.exe", "safari.exe")
  AND FileName IN ("powershell.exe", "cmd.exe", "wscript.exe", "cscript.exe",
    "mshta.exe", "rundll32.exe", "regsvr32.exe", "certutil.exe", "curl.exe",
    "wget.exe")
| stats count AS SpawnCount, values(CommandLine) AS CommandLines,
    earliest(timestamp) AS FirstSeen, latest(timestamp) AS LastSeen
    by ComputerName, UserName, ParentBaseFileName, FileName
| sort -SpawnCount
> **Analyst note:** This targets the social-engineering delivery vector described in the CVE — exploit requires a legitimate user to click a malicious link [1][2]. Management interface users are the likely target population. Review `CommandLines` values carefully for credential harvesting or lateral movement patterns.

### Query 2 — Anomalous Outbound Byte Volumes from Management Network Segment

nfdump -r /var/log/netflow/nfcapd.current \
  -f 'proto tcp and src net 10.0.0.0/24 and dst port 443' \

### Query 3 — Short-Duration, Low-Packet Flows Indicative of Token Beacon Callbacks
*Targets: Small, structured outbound sessions consistent with a stolen token being replayed or reported to an attacker-controlled server.*

## Hunting Notes

### What to Look For
- **Primary vector:** The exploit requires a legitimate PAN-OS management interface user to click a malicious link [1][2]. Hunt activity should focus on **management workstation users** and their browser behaviour, not the PAN-OS device itself as a network source.
- **Session token abuse:** Following token capture, look for authentication events to the PAN-OS management interface from **unexpected source IPs or geolocations** — this is the most actionable downstream indicator.
- **Referrer anomalies:** If web proxy logs are available, search for PAN-OS management interface URLs (`/php/`, `/login.php`, common PAN-OS management paths) appearing in HTTP Referer headers of requests going to external destinations.
- **No PoC available:** As of the time of this query generation, no public proof-of-concept is known [2]. Observable
