## Hunting Context

**Vulnerability Behaviour:** WSUS deserialization of untrusted data enabling unauthenticated RCE.
WSUS operates over HTTP/HTTPS (default ports 8530/8531). Exploitation typically involves
crafting a malicious serialised object delivered to the WSUS endpoint, potentially leading
to SYSTEM-level code execution on the WSUS server itself. Post-exploitation activity
commonly includes lateral movement, payload staging, and persistence mechanisms launched
from the WSUS service process (`WUSSyncServer.exe`, `w3wp.exe` under WSUS app pool).

## CrowdStrike Event Search Queries

### Query 1 — Suspicious Child Processes Spawned by WSUS Service Processes

Targets post-exploitation process execution: shells, loaders, or recon tools
launched from WSUS-associated IIS worker or service processes.

event_simpleName IN ("ProcessRollup2", "SyntheticProcessRollup2")
  ParentBaseFileName IN ("w3wp.exe", "WUSSyncServer.exe", "WUSServer.exe", "wsusservice.exe")
  FileName IN (
    "cmd.exe", "powershell.exe", "pwsh.exe", "wscript.exe", "cscript.exe",
    "mshta.exe", "rundll32.exe", "regsvr32.exe", "certutil.exe", "bitsadmin.exe",
    "curl.exe", "wget.exe", "net.exe", "net1.exe", "nltest.exe", "whoami.exe",
    "wmic.exe", "schtasks.exe", "at.exe", "msiexec.exe", "expand.exe"
  )
| stats count, values(CommandLine) as CommandLines,
         values(UserName) as Users,
         values(ParentBaseFileName) as ParentProcess
    by ComputerName, FileName, SHA256HashData
| where count >= 1
| sort -count

### Query 2 — Outbound Network Connections from WSUS Service Processes

Targets C2 callback or payload staging initiated by a compromised WSUS process.
WSUS processes should not initiate arbitrary outbound connections to non-Microsoft infrastructure.

event_simpleName="NetworkConnectIP4"
  ImageFileName IN (
    "*\\w3wp.exe",
    "*\\WUSSyncServer.exe",
    "*\\WUSServer.exe",
    "*\\wsusservice.exe",
    "*\\cmd.exe",
    "*\\powershell.exe"
  )
  RemotePort IN ("80", "443", "4444", "8080", "8443", "1337", "9001")
  NOT RemoteIP IN (
    "20.0.0.0/8",
    "13.0.0.0/8",
    "52.0.0.0/8",
    "40.0.0.0/8"
  )
| stats count,
         values(ImageFileName) as ProcessImage,
         values(RemotePort) as Ports,
         values(LocalIP) as LocalAddresses
    by ComputerName, RemoteIP, UserName
| sort -count

> **Analyst Note:** The `NOT RemoteIP` exclusions approximate Microsoft Azure/Update
> ranges. Replace with your organisation's validated Microsoft IP allowlist.
> Flag any WSUS process connecting to non-approved external IPs as HIGH priority.

### Query 3 — Suspicious File Writes by WSUS Processes (Webshell / Dropper Staging)

Targets file system artefacts: webshell drops, staged executables, or scripts
written to WSUS web directories or temp paths by the WSUS IIS worker process.

event_simpleName IN ("PeFileWritten", "ScriptFileWritten", "NewExecutableRenamed")
  ContextBaseFileName IN ("w3wp.exe", "WUSSyncServer.exe", "WUSServer.exe")
| eval FilePath=TargetDirectoryName + "\\" + TargetFileName
| where
    match(FilePath, "(?i)(\\\\inetpub\\\\|\\\\WSUSContent\\\\|\\\\UpdateServicesPackages\\\\|\\\\temp\\\\|\\\\windows\\\\temp\\\\)")
    OR match(TargetFileName, "(?i)\\.(aspx|ashx|asp|php|jsp|ps1|bat|vbs|js|hta|dll|exe)$")
| stats count,
         values(FilePath) as WrittenPaths,
         values(SHA256HashData) as Hashes,
         values(UserName) as Users
    by ComputerName, ContextBaseFileName
| sort -count

## nfdump Netflow Queries

### Query 1 — Inbound Exploit Attempts to WSUS Ports from External Hosts

Targets external hosts sending traffic to WSUS HTTP/HTTPS ports (8530/8531),
which should only receive connections from managed internal endpoints.

nfdump -r /var/log/netflow/nfcapd.current \
  -f 'proto tcp and (dst port 8530 or dst port 8531) and not src net 10.0.0.0/8 and not src net 172.16.0.0/12 and not src net 192.168.0.0/16' \
  -o 'fmt:%ts %te %sa %da %sp %dp %pkt %byt %fl %lbl' \
  -s record/bytes \
  -n 50

> **Analyst Note:** WSUS ports should receive traffic ONLY from internal managed clients
> and upstream Microsoft update servers. Any external source IP is anomalous and warrants
> immediate escalation. Replace RFC1918 blocks with your actual internal CIDR ranges.

### Query 2 — Anomalous Outbound Traffic Volume from WSUS Server Infrastructure

Targets data exfiltration or C2 beaconing from WSUS hosts — unexpected high-volume
or high-frequency outbound flows to non-Microsoft destinations.

nfdump -r /var/log/netflow/nfcapd.current \
  -f 'proto tcp and src ip x.x.x.x and not (dst net 20.0.0.0/8 or dst net 13.0.0.0/8 or dst net 52.0.0.0/8 or dst net 40.0.0.0/8) and (dst port 80 or dst port 443 or dst port 4444 or dst port 8080 or dst port 8443)' \
  -A srcip,dstip,dstport \
  -o 'fmt:%sa %da %dp %pkt %byt %fl' \
  -s record/bytes \
  -n 30

> **Analyst Note:** Replace `x.x.x.x` with the IP address(es) of your WSUS server(s).
> Replace Microsoft IP exclusions with your validated allowlist. Flows exceeding
> 10MB to a single external destination from a WSUS host should be escalated immediately.
