## CrowdStrike Event Search

### Query 1 — Inbound Network Connections Targeting LSVPN Gateway Ports
LSVPN satellite-to-GlobalProtect gateway communications typically traverse TCP/4501 (IPsec NAT-T) and UDP/500 (IKE). Anomalous sources initiating these connections may indicate exploitation attempts [1][2].

event_simpleName IN ("NetworkConnectIP4", "NetworkConnectIP6")
| where RemotePort IN (500, 4500, 4501)
| where LocalAddressIP4 != "" OR LocalAddressIP6 != ""
| stats count AS connection_count,
        dc(RemoteAddressIP4) AS unique_remote_ips,
        values(RemoteAddressIP4) AS remote_ips,
        earliest(ContextTimeStamp_decimal) AS first_seen,
        latest(ContextTimeStamp_decimal) AS last_seen
  by ComputerName, LocalPort, RemotePort
| where unique_remote_ips > 5
| sort -connection_count
// ANALYST NOTE: Tune unique_remote_ips threshold for your environment.
// High counts from unexpected source CIDRs may indicate reconnaissance
// or unauthenticated LSVPN connection attempts per CVE-2026-0283 mechanics [2].

### Query 2 — Post-Authentication Process Execution on PAN-OS Hosts
Following an authentication bypass [2], an attacker with an established VPN tunnel may pivot internally. Hunt for suspicious process trees spawned from VPN-related parent processes on systems acting as LSVPN gateways.

event_simpleName IN ("ProcessRollup2", "SyntheticProcessRollup2")
| where ParentBaseFileName IN ("openvpn", "iked", "vpnd", "pan_gp_hipreport", "GlobalProtect")
   OR FileName IN ("bash", "sh", "python3", "python", "perl", "nc", "ncat", "curl", "wget", "chmod")
| stats count AS exec_count,
        values(CommandLine) AS command_lines,
        values(FileName) AS child_processes,
        earliest(ContextTimeStamp_decimal) AS first_seen
  by ComputerName, UserName, ParentBaseFileName, ParentProcessId
| where exec_count > 0
| sort -exec_count

event_simpleName IN ("NetworkConnectIP4", "NetworkConnectIP6")

| lookup local://asset_inventory ComputerName OUTPUT asset_role

## nfdump Netflow Queries

### Query 1 — Traffic to LSVPN Exploit-Relevant Ports from Unexpected Sources
Targets IKE (UDP/500) and NAT-T (UDP/4500/4501) flows to hosts identified as LSVPN gateways. Unexpected source CIDRs may indicate unauthenticated connection attempts [2].

nfdump -r /var/log/netflow/nfcapd.current \
  -f 'proto udp and (dst port 500 or dst port 4500 or dst port 4501) \
      and dst ip 192.0.2.0/24' \
  -o 'fmt:%ts %td %sa %da %sp %dp %pkt %byt %fl' \
  -s record/packets -n 50
# ANALYST NOTE: Replace 192.0.2.0/24 with your LSVPN gateway CIDRs.
# CVE-2026-0283 is network-exploitable (AV:N) [1]; filter out known
# satellite peer CIDRs to surface anomalous sources.
# No specific attacker IPs are published — substitute x.x.x.x when IoCs emerge.
