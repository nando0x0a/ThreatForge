## Notes

1. **No public PoC known** — Exploitation has not been publicly demonstrated. IoC confidence is consequently LOW for all indicators above. Behavioural patterns are inferred from the vulnerability description rather than observed attack telemetry. [2]

2. **Attack surface prerequisites** — Successful exploitation requires: (a) OpenJDK installed from Ubuntu's `openjdk-25` package, (b) `mailcap` installed, (c) `xdg-desktop-portal-gtk` active (common on GNOME/GTK desktop environments), and (d) attacker ability to deliver a `.jar` file and trigger its MIME handler via the OpenURI portal. [2] This limits the realistic attacker population but makes desktop Ubuntu users a specific at-risk group.

3. **CVE age** — This CVE is 5 days old [as of generation date]. IoC landscape is likely incomplete. Analysts should re-query threat intel platforms (VirusTotal, MISP, OpenCTI, Shodan) as exploitation reports emerge.

4. **Recommended detection focus** — In the absence of network IoCs, detection should prioritise:
   - Process lineage monitoring for Java spawned from portal/snap processes
   - Filesystem monitoring for executable-flagged `.jar` files in user download paths
   - Audit logs for `xdg-open` or `mailcap` invocations of `.jar` files
