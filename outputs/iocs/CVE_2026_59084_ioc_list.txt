## Observable Behaviour Patterns
*Given the absence of confirmed IoCs at this early stage (CVE age: 6 days [1]; no public PoC known), the following behavioural patterns should be monitored as proxy indicators:*

**1. Cluster Communication Anomalies (Confidence: MEDIUM)**
- Unexpected or malformed inter-node cluster replication traffic on Tomcat cluster ports (default: TCP 4000 / UDP multicast, or custom ports per deployment).
- Connections to cluster replication ports from unexpected source IPs (i.e., outside the known cluster member IP range).
- Sudden spike in cluster replication traffic volume, which may indicate exploit attempts targeting misconfigured EncryptInterceptor.

**2. Authentication/Encryption Bypass Indicators (Confidence: MEDIUM)**
- Unencrypted or weakly encrypted cluster session replication traffic where EncryptInterceptor is expected to enforce encryption — detectable via packet inspection on cluster ports.
- Cluster communication accepted without a valid encryption key, indicating EncryptInterceptor is absent or misconfigured.

**3. Unexpected Process or Session Behaviour (Confidence: LOW)**
- New or unexpected Java process spawns from Tomcat worker threads (relevant given RCE classification [1]).
- Anomalous session replication data containing serialised Java objects (flag for Java deserialization inspection tooling).
- Unexpected outbound connections originating from Tomcat JVM processes.

**4. Configuration Indicators (Confidence: HIGH — defensive)**
- Presence of `<Interceptor className="org.apache.catalina.tribes.group.interceptors.EncryptInterceptor" ...>` in `server.xml` WITHOUT a strong, explicitly set encryption key → confirms misconfiguration exposure per advisory [2][3].
- Absence of EncryptInterceptor entirely in cluster configurations where replication is enabled → confirms exposure.

**5. Log Indicators (Confidence: MEDIUM)**
- Tomcat catalina.out / cluster log entries referencing `EncryptInterceptor` errors, key negotiation failures, or unexpected plaintext cluster messages.
- Access log entries showing high-frequency requests consistent with automated exploitation attempts against cluster endpoints.
