**Key Caveats:**

1. **PoC Repositories Not Fully Analysed:** The two named PoC repositories [6][7] (`wp2shell.git` from Crypto-Cat and `wp2shell` from h4cd0c) have not been reverse-engineered in this output. Analyst review of those repositories under organisational policy may yield specific payloads, User-Agent strings, and network indicators with higher confidence. **Do not execute PoC code in production environments.**

2. **Exploit Condition:** Exploitation requires untrusted input to be passed by an installed plugin or theme to `WP_Query` via the `author__not_in` parameter [3]. The attack surface is therefore installation-specific. Prioritise WordPress instances with high plugin/theme counts or those accepting user-controlled query parameters.

3. **Affected Versions:** WordPress 6.8.x < 6.8.6, 6.9.x < 6.9.5, and 7.0.x < 7.0.2 [3][4]. Patch immediately given KEV status [2].

4. **Active Exploitation:** CISA KEV listing [2] confirms in-the-wild exploitation. With 19 public PoCs [6][7] and a 3-day-old vulnerability, assume opportunistic mass scanning is already occurring. Defenders should treat any unpatched instance as compromised until forensically cleared.

5. **RCE Path:** While the primary vulnerability is SQL injection, the RCE flag (AV:N/PR:N/UI:N) [1] suggests a path from SQLi to code execution exists — potentially via reading/writing PHP files through SQL `INTO OUTFILE`, extracting credentials, or achieving authentication bypass leading to further exploitation. Monitor for secondary post-exploitation indicators (new admin accounts, file modifications, webshell uploads).
