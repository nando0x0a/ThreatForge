## Observable Behaviour Patterns
*Because no public PoC exists and the advisory context did not yield specific network or file indicators, the following behavioural patterns should be used in lieu of hard IoCs:*

1. **Unauthorized or anomalous Ingress/CRD writes** — Kubernetes audit log events showing `CREATE` or `UPDATE` operations on `Ingress` objects or NGINX CRDs (e.g., `VirtualServer`, `VirtualServerRoute`, `TransportServer`) from unexpected service accounts or users [1][2].

2. **Unexpected NGINX directive injection patterns** — Review of rendered `nginx.conf` inside the ingress-controller pod for directives not originating from legitimate configuration sources. Suspicious patterns include:
   - `ssl_certificate`, `ssl_certificate_key` pointing to arbitrary filesystem paths
   - `proxy_pass` to unexpected upstream addresses
   - `lua_*` or `js_*` directives if not normally present
   - `include` directives referencing non-standard paths
   - Embedded newline characters (`\n`, `%0a`) within annotation values

3. **Annotation value anomalies** — Ingress annotations containing characters or sequences indicative of injection attempts:
   - Semicolons (`;`) embedded in annotation values
   - Curly braces (`{`, `}`) in free-text annotation fields
   - Newline sequences (`\n`, `\r\n`, `%0a`, `%0d`) in annotation values
   - Hash/comment characters (`#`) mid-value

4. **Privilege escalation within cluster** — Accounts with `patch`/`update` RBAC permissions on `ingresses` or NGINX CRD resources that should not have write access [2].

5. **NGINX worker process spawning unexpected child processes** — Process tree anomalies under the NGINX master process inside the ingress-controller pod.

6. **Unexpected outbound connections from ingress-controller pod** — Egress to IPs/domains not matching defined upstreams, potentially indicating injected `proxy_pass` or `auth_request` directives.
