## Notes

### Context and Caveats

1. **Source limitations:** The advisory context provided [3] contained non-advisory content (JavaScript/browser-detection code). The technical IoC derivation relies primarily on the CVE description [1][5], the SLCyber research post [4], and tooling references [6][7]. Analyst should retrieve the full GHSA advisory at [3] for additional technical detail.

2. **Active exploitation confirmed:** CISA has listed CVE-2026-63030 in the Known Exploited Vulnerabilities catalog [2], and 86 public PoCs are known to exist [6][7]. This significantly elevates the confidence that the URI patterns above are being actively scanned for in the wild.

3. **RCE consequence:** Successful exploitation leads to remote code execution [1][4]. Post-exploitation IoCs to hunt for include:
   - Unexpected PHP files written to `wp-content/uploads/`, plugin, or theme directories (webshells)
   - New administrator accounts created via WP REST API or database
   - Outbound connections from the web server process to unfamiliar hosts
   - Scheduled task or cron additions in WordPress (`wp_options` table, `wp_cron`)

4. **No fabricated indicators:** IP addresses, domains, and file hashes are **not** present in the available sources. They have been deliberately omitted rather than fabricated. Analysts should enrich this list from:
   - CISA KEV supplemental advisories [2]
   - VirusTotal / URLhaus / Shodan queries against the URI patterns
   - Internal WAF/SIEM telemetry
   - Threat intel platform pivots on the `wp2shell` tooling references [6][7]

5. **Version scope:** Affected versions are WordPress 6.9.x < 6.9.5 and 7.0.x < 7.0.2 [1][5]. URI-based IoCs apply only where unpatched instances are confirmed; patched instances should still be monitored for post-exploitation persistence artefacts.

6. **Priority:** This CVE carries a Priority Score of 150 with tags [KEV][RCE][CRIT][T1][POC] — treat all unpatched WordPress instances in scope as **critically at risk**.
