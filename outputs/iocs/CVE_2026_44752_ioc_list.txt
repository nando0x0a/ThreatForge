## Observable Behaviour Patterns
*(In absence of confirmed IoCs, the following behavioural patterns should be used for detection and hunting)*

1. **Stored XSS injection via URL**: HTTP requests to SAP NetWeaver AS Java endpoints containing JavaScript payloads within URL parameters (e.g., `<script>`, `javascript:`, `onerror=`, `onload=`, encoded variants `%3Cscript%3E`, `%6A%61%76%61%73%63%72%69%70%74%3A`) [1][4].

2. **Cookie-stealing payloads**: The advisory context [2] shows `document.cookie` manipulation and `location` redirection — defenders should monitor for outbound requests from browser sessions to unexpected external domains carrying cookie data, which is consistent with stored XSS exploitation.

3. **OAuth redirect abuse**: Redirection chains through SAP BTP OAuth endpoints (`eu11.hana.ondemand.com`) following access to crafted portal URLs [2] — anomalous OAuth `code` exchanges not initiated by legitimate user sessions.

4. **Unauthenticated access preceding victim redirect**: Access log entries showing unauthenticated requests to NetWeaver Java portal paths immediately followed by victim browser outbound connections to non-SAP domains [1].

5. **Encoded JavaScript in GET parameters**: URL parameters containing HTML/JS encoded strings delivered to NetWeaver AS Java endpoints — look for `encodeURIComponent`-style encoding patterns in web access logs [2].
