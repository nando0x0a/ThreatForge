## HTTP Header Indicators

**Confidence: HIGH** — Directly described in the vulnerability mechanism [2][4].

The following request headers are the core exploitation mechanism. Requests that set these headers unexpectedly, or where `X-User-Id` does not match the authenticated session, warrant investigation:

HEADER: X-User-Id          # Attacker-supplied header used to impersonate another user [2]
HEADER: X-Session-Id       # Unencoded/plaintext session_id forwarded to terminal backend [2]

## Observable Behaviour Patterns
*(In lieu of concrete IoCs, the following behavioural patterns should be used for detection and triage.)*

1. **Header Manipulation**: HTTP requests to the Open WebUI terminal backend containing a `X-User-Id` header value that does not correspond to the authenticated user's session [2][4].

2. **Session ID Exposure**: Presence of unencoded `session_id` values in request headers or logs forwarded to the terminal backend component [2].

3. **Cross-User Terminal Access**: A single source IP or session accessing terminal functionality associated with multiple distinct user IDs in a short timeframe.

4. **Unexpected Terminal API Calls**: Requests to the terminal backend originating from accounts or IPs with no prior terminal activity, particularly where `X-User-Id` is explicitly set in the request.

5. **Crafted Request Pattern**: Requests that explicitly set both `X-User-Id` and `X-Session-Id` (or `session_id`) headers simultaneously — normal browser-originated traffic is unlikely to manually set both [2].
