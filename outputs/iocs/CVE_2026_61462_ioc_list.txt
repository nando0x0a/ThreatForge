### Observational Behaviour Patterns

In the absence of confirmed active-exploitation IoCs, analysts should monitor for the following **behavioural patterns** in logs and telemetry:

1. **Anomalous job_id values** — Any `job_id` field in requests to the mcp-gitlab service containing `..`, `/`, `%2e`, `%2f`, `%252e`, or similar traversal sequences [2].
2. **Unexpected GitLab API endpoint access** — GitLab API calls originating from the mcp-gitlab service to endpoints outside the expected `/api/v4/projects/*/jobs/*` scope, particularly administrative or secrets-related endpoints [2].
3. **Unauthenticated requests to the MCP server** — Given AV:N/PR:N/UI:N [1][4], monitor for requests from external or unexpected source IPs.
4. **Elevated API activity** — Unusual volume of GitLab API calls via mcp-gitlab, potentially indicating automated exploitation attempts.

### Additional Context

- CVE is 6 days old [1]; no public PoC is currently known, but the vulnerability is described in sufficient detail in the advisory [2] that independent PoC development is plausible in the near term.
- The affected file `build/index.js` is part of the `mcp-gitlab` npm package; analysts should verify installed package versions against the advisory [2][3].
