## Behavioural Patterns
*In lieu of specific IoCs, the following observable behaviours are relevant for detection and hunting:*

- **Stored XSS payload injection:** Authenticated HTTP POST requests to GitLab user-controlled content fields (comments, descriptions, wikis, snippets, profiles) containing unsanitized script content. Look for `<script>`, `javascript:`, `onerror=`, `onload=`, `data:text/html` or equivalent encoding variants in request bodies to GitLab API or web endpoints [1].
- **Cross-user session activity:** Unusual JavaScript execution triggered in victim browser sessions; may appear as out-of-character API calls originating from victim accounts shortly after viewing attacker-controlled content.
- **Authentication prerequisite:** Exploitation requires an authenticated session [1]. Unauthenticated requests to content-injection endpoints should be filtered; focus hunting on authenticated (session-cookie or PAT-bearing) requests containing script-like patterns.
- **Affected version range:** GitLab CE/EE versions ≥ 15.7 and < 18.11.7, 19.0.x < 19.0.4, and 19.1.x < 19.1.2 [1][2]. Log sources from instances in these ranges warrant elevated scrutiny.
- **No public PoC known** as of generation date; exploitation is likely lower-volume and targeted at this stage.
