## Observable Behaviour Patterns
*(In lieu of confirmed technical IoCs, the following behavioural patterns are relevant to a local privilege escalation via broken access control on Exchange Server [1][2]:)*

1. **Unexpected privilege elevation events** — Authenticated local users or service accounts gaining Exchange Organization Management or higher role group membership without administrative action.
2. **Abnormal Exchange role assignment changes** — New role assignments via Exchange Management Shell (`New-ManagementRoleAssignment`) by non-privileged accounts.
3. **Lateral movement from Exchange host** — Processes spawned under Exchange service accounts (e.g., `NETWORK SERVICE`, `MSExchangeOWAAppPool`) executing unexpected commands.
4. **Local access from non-admin accounts** — Interactive or remote (RDP/WinRM) logons to Exchange Server nodes by accounts that do not ordinarily require local access.
5. **Security audit log tampering** — Attempts to clear Windows Security Event Log on Exchange hosts following access control abuse.
