## Notes

### Confidence Assessment
- **Overall IoC confidence: LOW** — No public proof-of-concept is known, no active exploitation has been reported, and the advisory [2] does not disclose specific technical exploit strings or indicators. IoCs listed above are behavioural/pattern-based, derived from the vulnerability class description.

### Vulnerability Context
- CVE-2026-20297 affects Splunk Enterprise < 10.4.1, 10.2.5, 10.0.8, 9.4.13, 9.3.14 and Splunk Cloud Platform < 10.5.2605.0, 10.4.2604.6, 10.2.2510.18, 10.1.2507.24 [1][2].
- The vulnerability is a **path traversal in the app installation workflow** that fails to restrict installation paths to the intended app directory [2].
- Exploitation requires an authenticated user with **`edit_local_apps` AND `install_apps` capabilities** [1][2]. This is a significant constraint — it limits the attack surface to privileged Splunk roles.
- CVSS Score: **7.2 (HIGH)** [1]. The score reflects the file-write impact available to a privileged attacker.
- CVE is 4 days old with no known public PoC. **Priority Tier: MONITOR.**

### Observable Behaviour Patterns (in lieu of confirmed IoCs)
Analysts should hunt for the following behaviours in Splunk environments:

1. **Unexpected file writes** outside `$SPLUNK_HOME/etc/apps/` or `$SPLUNK_HOME/etc/peer-apps/` directories, particularly following app installation events.
2. **App installation events** (`install_apps` capability actions) by accounts not expected to perform this operation — review `_audit` index for `action=install_app`.
3. **New or modified files** in sensitive directories such as `$SPLUNK_HOME/bin/`, OS system paths, or web-accessible directories, correlated with app installation timestamps.
4. **Role/capability assignments** of `edit_local_apps` or `install_apps` to unexpected accounts (lateral privilege movement pre-exploitation).
5. **Splunk REST API calls** to app install endpoints from unusual source IPs or at unusual hours.

### Recommended Splunk Search (Draft — analyst review required)
index=_audit action=install_app
| eval traversal_flag=if(match(app_name, "\.\./|%2e%2e|%252e"), "SUSPECTED_TRAVERSAL", "clean")
| where traversal_flag="SUSPECTED_TRAVERSAL"
| table _time, user, app_name, src_ip, traversal_flag
