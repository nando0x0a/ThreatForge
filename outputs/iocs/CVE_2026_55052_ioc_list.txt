### Observable Behaviour Patterns (in lieu of specific IoCs)
Since no confirmed IoCs are available, analysts should monitor for the following behavioural patterns consistent with exploitation of a SharePoint broken access control / missing authorization check vulnerability [1][2]:

1. **Privilege Escalation Events:** Authenticated users (particularly low-privilege accounts) gaining elevated SharePoint permissions unexpectedly — e.g., Site Collection Administrator rights granted without administrative action.
2. **Anomalous API Calls:** High volumes of SharePoint REST API (`/_api/`) or CSOM requests from accounts that do not normally make such calls, particularly targeting permission management endpoints.
3. **Authorization Bypass Patterns:** HTTP 200 responses to SharePoint endpoints that would normally return 403/401 for the requesting account's permission level.
4. **Lateral Movement Indicators:** Accounts accessing SharePoint sites or document libraries outside their normal access scope, especially across site collections.
5. **Audit Log Anomalies:** SharePoint Unified Audit Log entries showing permission changes, role assignments, or site access events attributed to accounts without corresponding administrative sessions.
