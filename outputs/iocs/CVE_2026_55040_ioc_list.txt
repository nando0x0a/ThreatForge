### Vulnerability Profile
- CVE-2026-55040 describes a **broken authentication vulnerability in Microsoft Office SharePoint** [1][3], with a CVSS score of **9.1 CRITICAL** [1], attributed to weak authentication mechanisms permitting unauthenticated network-based bypass.
- Attack vector is **network-exploitable with no privileges and no user interaction required** (AV:N/PR:N/UI:N) [1], meaning exploitation does not require prior authentication — any exposed SharePoint instance is a potential target.
- RCE capability is confirmed in the metadata, making this a high-priority detection target even without confirmed IoCs.

### Recommended Observable Behaviour Patterns (in lieu of confirmed IoCs)
Given the authentication bypass + RCE profile, analysts should hunt for:

1. **Unauthenticated or anomalously authenticated requests** to SharePoint REST API or layout endpoints from external IP ranges.
2. **HTTP 200 responses to SharePoint endpoints** that should require authentication, particularly from IPs with no prior session history.
3. **Unusual POST requests** to SharePoint authentication endpoints without valid session tokens.
4. **Process spawning from SharePoint worker processes** (e.g., `w3wp.exe` spawning `cmd.exe`, `powershell.exe`, `mshta.exe`, or `wscript.exe`) — indicative of post-exploitation RCE.
5. **Outbound connections from SharePoint servers** to external IPs, particularly on non-standard ports.
6. **Failed-then-succeeded authentication sequences** against SharePoint at high volume from a single source — indicative of auth bypass probing.
