## Observable Behaviour Patterns
*In lieu of confirmed IoCs, the following behavioural patterns are consistent with exploitation of an insecure deserialization privilege escalation vulnerability on Microsoft Exchange Server [1][2][3]:*

- **Anomalous Exchange process spawning:** Child processes (e.g., `cmd.exe`, `powershell.exe`, `wscript.exe`) spawned from Exchange worker processes such as `MSExchangeServiceHost.exe`, `w3wp.exe` (Exchange application pools), or `EdgeTransport.exe` at elevated privilege levels.
- **Deserialization-related exceptions in Exchange logs:** Errors or stack traces in Exchange diagnostic logs referencing .NET deserialization namespaces (`System.Runtime.Serialization`, `BinaryFormatter`, `ObjectStateFormatter`) from unexpected or unauthenticated code paths.
- **Unexpected privilege escalation by authenticated Exchange accounts:** Authenticated users or service accounts suddenly acquiring SYSTEM or Exchange Organization permissions without corresponding administrative action.
- **Unusual .NET assembly loads:** Arbitrary .NET assemblies loaded into Exchange worker processes from non-standard directories (e.g., `%TEMP%`, user-writable paths), detectable via ETW or Sysmon Event ID 7.
- **Access to sensitive Exchange configuration data post-exploitation:** Unusual reads of Exchange configuration databases or Active Directory objects by accounts not previously associated with such access.
- **Lateral movement from Exchange host:** Post-exploitation activity originating from the Exchange server host toward domain controllers or other internal systems.
