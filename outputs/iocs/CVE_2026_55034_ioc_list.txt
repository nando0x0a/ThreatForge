### Confidence Assessment
- **Overall IoC confidence: LOW.** The vulnerability is 5 days old [1][2], no public proof-of-concept is known, and there is no reported active exploitation or CISA KEV listing for this CVE.
- The Microsoft Security Response Center advisory context [2] did not yield parseable technical detail (JavaScript required; content not rendered), preventing extraction of specific attack paths, payload signatures, or affected endpoints.
- NVD [1] and CVE.org [3] records confirm the vulnerability class (stored XSS / spoofing) but do not document observable IoCs at this stage.

### Vulnerability Context for Detection Framing
- **Type:** Stored Cross-Site Scripting (XSS) [1] — attacker must be **authorized** (authenticated) to inject payload [1][2].
- **Impact:** Spoofing over a network — likely session hijacking, credential capture, or UI redirection via malicious script execution in victim browser context.
- **CVSS:** 7.3 HIGH [1] — network attack vector, low complexity assumed.
- **Attacker Prerequisite:** Authorization required; limits drive-by exploitation risk. Threat model prioritizes **insider threat, compromised accounts, or OAuth abuse** as initial access paths.

### Observable Behaviour Patterns (in lieu of confirmed IoCs)
1. **Unusual script tags or encoded JavaScript** (`<script>`, `%3Cscript%3E`, `javascript:`) appearing in SharePoint page content, list fields, or web part properties.
2. **Unexpected outbound requests** from SharePoint-rendered pages to external domains (exfiltration of cookies/tokens).
3. **SharePoint audit logs** showing content edits by authorized users followed shortly by access from unrelated accounts or IP ranges.
4. **HTTP responses** from SharePoint containing unsanitized `<script>` blocks or `onerror`/`onload` event handlers in rendered HTML — detectable via proxy/CASB inspection.

### Recommended Analyst Actions
- Monitor Microsoft MSRC advisory [2] for patch release and technical detail update.
- Cross-reference with SharePoint access logs for anomalous content modification by authorized accounts.
- Apply Microsoft security update when published; patch status not confirmable from available advisory rendering [2].
