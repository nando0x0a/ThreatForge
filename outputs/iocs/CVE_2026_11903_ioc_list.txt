## Observable Behaviour Patterns
*Given no public PoC [contextual metadata] and no confirmed active exploitation, the following behavioural patterns represent the most actionable detection focus:*

1. **Stored XSS Injection Phase**
   - POST requests to MOVEit Transfer Ad Hoc module endpoints [1][2] containing HTML/script injection payloads (e.g., `<script>`, `<img onerror=`, `javascript:`, SVG-based vectors) in user-controllable fields (message body, subject, filename metadata, recipient fields).

2. **Payload Execution Phase (requires victim interaction)**
   - Authenticated user loads an Ad Hoc Transfer page containing stored malicious content.
   - Browser-side script execution: outbound requests from end-user browsers to unexpected external domains (callback/exfiltration), unexpected cookie or credential capture behaviour.
   - Session token theft indicators: re-use of session tokens from anomalous source IPs following Ad Hoc module access.

3. **Post-Exploitation Patterns**
   - Unusual authenticated API activity following Ad Hoc module access (file downloads, user enumeration, permission changes) — consistent with session hijacking via XSS.
   - New administrative accounts or modified permissions following Ad Hoc module interactions.
