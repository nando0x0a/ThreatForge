### Confidence Summary

- URL indicators: HIGH — validated against public advisory and vendor documentation
- User-agent indicators: MEDIUM — non-unique; useful only in combination with URI patterns

### Affected Versions
- Apache HTTP Server 2.4.49 and 2.4.50 ONLY
- CVE-2021-42013 specifically resolves the incomplete patch from CVE-2021-41773 (2.4.49)
- Apache 2.4.51 and later are NOT affected

### Exploitation Prerequisites
1. Apache 2.4.49 or 2.4.50 must be running
2. For RCE: CGI or mod_cgi must be enabled AND traversal path must reach executable
3. For file read: Target directory must not have `Require all denied` configured
4. Default Apache configurations on some distributions were vulnerable out-of-the-box

### Detection Guidance
- Detections on URI patterns alone may produce false positives; correlate with:
  - HTTP 200 responses to traversal URIs (confirms vulnerable + accessible path)
  - POST body containing shell metacharacters (echo;, /bin/sh, whoami, id)
  - Anomalous child processes spawned by Apache (httpd → sh/bash/python)
- Server-side: Monitor for Apache spawning unexpected child processes
