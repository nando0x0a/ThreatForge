## Observable Behaviour Patterns
# Confidence: MEDIUM — derived from vulnerability description [1][3][4]

These behavioural patterns should be used in lieu of specific IoCs for detection engineering:

1. **Unexpected git.exe execution from repository/workspace directories**
   Cursor for Windows 3.2.16 automatically executes `git.exe` found within the opened workspace rather than resolving exclusively to a trusted system or PATH-controlled binary [1]. Monitor for `git.exe` process launches where the image path resolves to a workspace or user-controlled directory (e.g., `%USERPROFILE%\`, `.\`, or any repository clone path) rather than a known-good installation path such as `C:\Program Files\Git\cmd\git.exe` or equivalent.

2. **Child process spawned by Cursor from non-standard git.exe path**
   Alert on `cursor.exe` (or related Cursor process) spawning a child process named `git.exe` from any path other than a trusted git installation directory.

3. **Crafted repository opened in Cursor**
   Social engineering vector: a victim is induced to open a crafted repository in Cursor [1]. Correlate repository clone/open events with subsequent unexpected process spawning.

4. **Binary written to workspace root matching system binary names**
   Detect creation of executables (`.exe`, `.dll`) in repository working directories that share names with system or well-known developer tool binaries (`git.exe`, `git.dll`, etc.).
