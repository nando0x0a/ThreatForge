## Notes

**Overall Confidence: LOW-MEDIUM** for all indicators above.

1. **No public PoC is known** at time of generation [3]. No exploit traffic, scanning campaigns, or threat-actor attribution has been observed or reported in the provided context.

2. **Vulnerability scope is broad** [1]: Affected versions span Apache Tomcat 8.5.0–8.5.100, 9.0.0.M1–9.0.119, 10.1.0-M1–10.1.56, and 11.0.0-M1–11.0.23. The attack surface is extremely wide for any internet-exposed Tomcat instance.

3. **Exploitation requires specific conditions** [1][2]: The RewriteValve must be configured and in use. Deployments without RewriteValve configured are not exposed to this specific bypass path. Analysts should prioritise identification of Tomcat deployments actively using RewriteValve rules.

4. **CVSS 9.1 CRITICAL, network-exploitable, no authentication required, no user interaction required** [1]. Given AV:N/PR:N/UI:N, opportunistic scanning should be anticipated rapidly once PoC becomes available.

5. **Recommended detection approach (observable behaviour patterns):**
   - Monitor web/application logs for requests to protected URI paths containing percent-encoded characters (`%2F`, `%2e`, `%25`, `%252F`) that reach backend resources without triggering expected HTTP 403/401 responses.
   - Alert on HTTP 200 responses to paths that security constraints should have blocked.
   - Correlate RewriteValve rule matches in Tomcat access logs against security constraint enforcement outcomes.
   - Monitor for anomalous access to admin interfaces (e.g., `/manager/`, `/host-manager/`, `/admin/`) via encoded URI variants.

6. **CVE is 6 days old** [1]. IoC landscape may evolve rapidly. Re-assess upon public PoC publication or first confirmed exploitation reports.
