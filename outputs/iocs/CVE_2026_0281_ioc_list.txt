## Observable Behaviour Patterns

*In the absence of confirmed IoCs, the following behavioural patterns are relevant for detection hypothesis development:*

1. **Session Token Reuse from Unexpected Sources** — Legitimate PAN-OS management sessions suddenly originating from IPs inconsistent with the authenticated user's expected location or ASN, potentially indicating a stolen token being replayed [1][2].

2. **User-Initiated Outbound Requests to External Domains** — If a legitimate admin user clicks a malicious link, HTTP referrer logs or proxy logs may show a management-plane-originating request to an external/unexpected domain immediately prior to anomalous session activity [1].

3. **Concurrent or Near-Simultaneous Sessions** — The same session token appearing in requests from two distinct source IPs within a short time window — one legitimate, one anomalous.

4. **Anomalous Management Interface Access Timing** — Authenticated management interface activity at unusual hours, particularly if preceded by no direct login event from the same session source IP.

5. **HTTP Referrer Anomalies** — Requests to the PAN-OS management interface with `Referer` headers pointing to external or unexpected domains, which may indicate a clickjacking or redirect-based token capture attempt consistent with the vulnerability description [1][3].
