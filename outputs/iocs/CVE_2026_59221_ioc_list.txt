### Observable Behaviour Patterns
In the absence of confirmed infrastructure IoCs, analysts should monitor for the following behavioural patterns in Open WebUI deployments running versions **0.9.6 up to (excluding) 0.10.0** [1]:

1. **HTTP requests to proxy-related endpoints** containing percent-encoded dot-dot sequences (`%2e%2e`, `%252e%252e`, `%2f`) in the path.
2. **Unexpected file system path access** in application logs — particularly paths outside the intended proxy scope (e.g., `/etc/`, `/proc/`, application credential files).
3. **Double or triple percent-encoded characters** in proxy path segments, indicative of sanitization bypass attempts.
4. **Repeated 200 OK responses** to malformed proxy paths that would normally be expected to return 400/403 errors on a patched system.
5. **PoC activity:** One public proof-of-concept is known to exist [5], which may increase opportunistic scanning activity. Monitor for automated scan patterns against Open WebUI proxy endpoints.

### Patch Reference
The fix is available in Open WebUI 0.10.0. See commit [2] and PR [3] for patch details. Analysts should cross-reference the patched `_sanitize_proxy_path` implementation to derive more precise endpoint paths for detection rule refinement.
