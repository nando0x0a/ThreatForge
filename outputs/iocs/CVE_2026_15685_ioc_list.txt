## Observable Behaviour Patterns
*Given the absence of a public PoC [2] and no known active exploitation, the following behavioural patterns are provided as detection guidance in lieu of hard IoCs:*

1. **Malformed or out-of-bounds array index values** in blob-related API requests targeting Ollama's HTTP API surface (default port 11434/tcp) [2].
2. **Sudden Ollama process crashes or service unavailability** following inbound API requests — indicative of a successful memory access past array bounds trigger [1][2].
3. **Unauthenticated, high-frequency requests** to blob endpoints from unexpected source IPs, consistent with DoS attempt patterns. Ollama requires no authentication by default [2].
4. **Network connections to Ollama's API port (default 11434/tcp) from external/untrusted sources** — Ollama is not designed for unauthenticated internet exposure; any such traffic is inherently anomalous [2].
5. **Anomalous process memory faults or OOM-killer events** in the Ollama process on Linux hosts.
