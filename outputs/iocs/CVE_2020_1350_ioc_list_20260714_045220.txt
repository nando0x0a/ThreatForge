## Observable Behaviour Patterns
*(Used in place of specific IoCs where confirmed indicators are unavailable)*

PATTERN-001 | Confidence: HIGH
  Description : Anomalously large DNS response packets received by Windows DNS Server
  Detail      : SIGRed is triggered via a crafted DNS response containing an oversized
                SIG record (>64KB). Look for DNS TCP responses where the RDLENGTH field
                causes integer overflow in dns.exe.
  Observable  : DNS TCP responses exceeding 65,535 bytes in length directed at
                Windows DNS Server (UDP/TCP port 53)
  Log Source  : Network flow data, IDS/IPS, DNS server debug logs

PATTERN-002 | Confidence: HIGH
  Description : DNS query forwarding abuse — attacker-controlled upstream resolver
  Detail      : Exploitation may involve manipulating DNS forwarding so the vulnerable
                server queries an attacker-controlled resolver that returns the
                malicious SIG record payload.
  Observable  : Windows DNS Server initiating TCP/53 connections to unexpected or
                external resolvers not in the approved forwarder list.
  Log Source  : DNS server debug logs, firewall egress logs, netflow

PATTERN-003 | Confidence: HIGH
  Description : dns.exe process anomalies post-exploitation
  Detail      : Successful RCE runs in the context of SYSTEM (Local System Account).
                Post-exploitation activity would originate from dns.exe.
  Observable  : - dns.exe spawning child processes (cmd.exe, powershell.exe, etc.)
                - dns.exe making outbound network connections on non-DNS ports
                - New services or scheduled tasks created under SYSTEM context
                shortly after DNS service activity
  Log Source  : Windows Event Log (Security, System), Sysmon Event IDs 1/3/7/11,
                EDR telemetry

PATTERN-004 | Confidence: MEDIUM
  Description : DNS service crashes or restarts preceding exploitation
  Detail      : Failed or probe exploitation attempts may cause dns.exe to crash.
  Observable  : Windows Event ID 1000 (Application Error) for dns.exe,
                unexpected DNS service restarts, WER crash dumps for dns.exe
  Log Source  : Windows Event Log (Application), WER logs (%LOCALAPPDATA%\CrashDumps)

PATTERN-005 | Confidence: MEDIUM
  Description : Inbound DNS-over-TCP from internet-facing sources
  Detail      : While DNS-over-TCP is legitimate, internet-exposed DNS servers
                receiving unsolicited TCP/53 sessions — especially with large
                payloads — should be treated as suspicious.
  Observable  : TCP/53 sessions initiated from external/untrusted IPs to
                Windows DNS Server infrastructure
  Log Source  : Firewall logs, network flow/PCAP, IDS/IPS

PATTERN-006 | Confidence: HIGH
  Description : Workaround registry key absence (pre-patch indicator)
  Detail      : Microsoft issued a registry-based workaround (TcpReceivePacketSize)
                prior to patch availability. Absence of this key on unpatched systems
                indicates active exposure.
  Observable  : HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\DNS\Parameters\
                TcpReceivePacketSize — value should be 0xFF00 (workaround) or
                patch KB applied.
  Log Source  : Registry audit, vulnerability scanner output, SCCM inventory

## Sigma / Detection Hint (Summary)

Focus detection on:
  - dns.exe spawning non-DNS child processes (Sysmon EID 1, parent: dns.exe)
  - dns.exe outbound connections on ports ≠ 53 (Sysmon EID 3)
  - Oversized DNS TCP payload alerts (IDS signature layer)
  - DNS service crash events (Windows EID 1000)
