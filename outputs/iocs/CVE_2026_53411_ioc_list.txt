### Vulnerability Context
- **Type:** TOCTOU (Time-Of-Check Time-Of-Use) race condition in Zoom Client for Windows installation and uninstallation routines [1][2].
- **Impact:** Local privilege escalation to a higher-privileged context; requires an authenticated local user [1].
- **CVSS:** 7.8 HIGH [1]. No CVSS discrepancy between sources was identified in the provided context; however, the advisory body could not be fully parsed [2], so analyst cross-check is recommended.
- **PoC:** No public proof-of-concept known at time of generation.
- **Age:** 3 days — patch adoption will be low; treat as active risk on unpatched Windows endpoints.
- **CISA KEV:** No KEV entry noted in provided context.

### Recommended Detection Approach (Behaviour-Based)
Since no confirmed IoCs exist, detection should focus on:
1. **Process integrity:** Monitor `ZoomInstaller.exe` / `ZoomUninstaller.exe` spawning unexpected child processes or writing to `System32` / privileged paths.
2. **File substitution:** Alert on file modifications within the Zoom installation directory during active installer/uninstaller execution windows (short time-window file replacement is characteristic of TOCTOU exploitation).
3. **Privilege changes:** Correlate `ZoomInstaller.exe` process tree with Windows Event ID 4672 (special privileges assigned) or 4688 (new process creation) showing elevated token grants to unexpected users.
4. **Signed binary abuse:** Monitor for Zoom-signed binaries executing from non-standard paths, which may indicate a planted binary winning a race condition.

Analyst action: Obtain the full ZSB-26013 advisory text directly from [2] to confirm affected installer versions and any vendor-provided hashes before operationalising these patterns.
