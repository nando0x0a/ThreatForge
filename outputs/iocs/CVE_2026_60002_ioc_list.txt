*No specific attacker IPs or infrastructure identified in the provided source material at this time.*

**Observable Network Behaviour (in lieu of confirmed IPs):**
- Unexpected or repeated SSH key re-exchange (`SSH_MSG_KEXINIT`) sequences from client to server [2][3]
- SSH sessions exhibiting anomalous re-exchange frequency, particularly where the server-side host key changes mid-session [1][2]
- SSH connections on TCP/22 (or non-standard SSH ports) from untrusted or external clients initiating rapid key re-exchange cycles
- Session crashes or unexpected SSH daemon terminations following re-exchange events — potential indicator of exploitation attempt [2][3]

### ⚠️ Severity Discrepancy Notice
The provided CVE metadata assigns a CVSS score of **7.7 (HIGH)** [1][4]. However, the vulnerability description confirms **network-exploitable RCE with AV:N/PR:N/UI:N**, which — depending on the specific CVSS vector breakdown — may warrant analyst review of whether the base score fully reflects exploitability in context. Analysts should review the full CVSS vector string at [1] and [4] independently and not rely solely on the numeric score for prioritisation. The Priority Tier assigned by the detection pipeline is **CRITICAL — ACT NOW** (Priority Score: 90), which exceeds what the 7.7 CVSS score alone might suggest; analysts should validate this gap.

### Exploit Mechanics (for detection context)
- Attack requires: (1) a client running OpenSSH < 10.4 connected to a server, and (2) the server changes its host key during an active key re-exchange [1][2][3]
- The use-after-free is **client-side** — the vulnerable component is the SSH client, not the server daemon [2]
- Detection focus should therefore be on **client-side OpenSSH versions** in the environment, not exclusively server exposure

### Recommended Detection Actions
1. Inventory all hosts running OpenSSH client < 10.4 [2]
2. Monitor SSH logs for unusual `kexinit` re-exchange sequences
3. Alert on sshd or ssh process crashes following re-exchange events
4. No confirmed in-the-wild exploitation IoCs are available as of this draft — monitor threat intelligence feeds for emerging indicators
