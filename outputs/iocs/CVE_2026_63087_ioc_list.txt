## Observable Behaviour Patterns
*In the absence of attacker IoCs, the following exploitation behaviours should be treated as detection targets:*

1. **Unauthenticated POST/GET requests to the plugin install endpoint** — CVE-2026-63087 requires no authentication (PR:N, UI:N) [1][4]. Any request to the plugin install URI lacking valid session credentials should be treated as suspicious.

2. **Requests using default/hardcoded `stack_id` and `org_id` parameter values** — The vulnerability is caused by hardcoded default values for these parameters [1][2]. Requests carrying predictable/default integer values (e.g., `stack_id=1`, `org_id=1` or similar) to the install endpoint warrant investigation [2][3].

3. **Unexpected `PluginAuthToken` issuance or admin-level API activity** — Post-exploitation, an attacker obtains a valid `PluginAuthToken` and may perform admin actions [1][2]. Monitor Grafana OnCall audit logs for unexpected plugin installations, configuration changes, or admin actions not correlated to authenticated user sessions.

4. **Anomalous traffic to Grafana OnCall from external/internet-facing sources** — Given AV:N (network-accessible, no authentication required) [1], any access to the plugin install endpoint originating from external or unexpected IP ranges is a high-confidence indicator of exploitation attempt.

5. **Repeated or rapid sequential requests to the plugin install endpoint** — Scanning or brute-force behaviour targeting the endpoint to confirm exploitability.
