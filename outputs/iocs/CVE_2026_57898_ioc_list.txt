## Observable Behaviour Patterns
*Because no public PoC is available and IoCs are sparse (6-day-old CVE [1]), focus detection on the following behavioural patterns:*

1. **Unauthenticated HTTP requests to the AAS thumbnail endpoint** [2]
   - PUT or POST requests to any `/aas/thumbnail` or equivalent path from unexpected source IPs, with no prior authentication exchange.
   - Confidence: MEDIUM

2. **Filename anomalies in thumbnail upload requests** [2]
   - Request payloads containing path traversal sequences in filename parameters: `../`, `..\`, URL-encoded variants (`%2F`, `%2E%2E`), null bytes, or absolute path prefixes.
   - Confidence: MEDIUM

3. **Unexpected file creation outside designated storage directories**
   - On MongoDB-backend BaSyx deployments [2], new files appearing in web root, system directories, or shell-accessible locations (e.g., cron directories, SSH authorized_keys paths) following thumbnail API activity.
   - Confidence: MEDIUM

4. **Abnormal outbound connections following file write**
   - Post-exploitation RCE activity [1] may manifest as unexpected outbound connections from the BaSyx server process if a web shell or scheduled task is written.
   - Confidence: LOW (no PoC, speculative)

5. **Large or non-image content in thumbnail API payloads**
   - Requests to the thumbnail endpoint carrying content-types inconsistent with images (e.g., `text/x-php`, `application/x-sh`, `text/plain`) or oversized payloads.
   - Confidence: MEDIUM

## Notes

- **Indicator confidence is generally LOW-to-MEDIUM** across all categories. CVE-2026-57898 is 6 days old [1], no public PoC exists, and no observed exploitation campaigns have been reported. IoCs above are derived from the vulnerability's technical nature (CWE-22, unauthenticated file write via AAS thumbnail API in MongoDB deployments) [2] rather than from confirmed attack telemetry.
- **CVSS 9.0 CRITICAL with AV:N/PR:N/UI:N and confirmed RCE potential** [1] means the attack surface is fully remote and unauthenticated. Treat any anomalous thumbnail API traffic as high-priority for triage.
- **Affected versions**: Eclipse BaSyx Java Server SDK 2.0.0-milestone-05 through 2.0.0-milestone-12 [2]. Detections should be scoped to deployments running these versions with MongoDB backends.
