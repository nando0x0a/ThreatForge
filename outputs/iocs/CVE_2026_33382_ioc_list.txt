## Observable Behaviour Patterns
> Given the absence of confirmed IoCs, analysts should hunt for the following behavioural patterns:

1. **Abnormally large HTTP POST/PUT request bodies** to Grafana API endpoints — requests significantly exceeding normal operational size (e.g., >10 MB, >50 MB) from a single source within a short window.

2. **Unauthenticated requests to Grafana API paths** — exploitation requires no authentication [1][2]; look for API calls to `/api/*` paths with no valid session token or API key.

3. **Grafana process memory exhaustion** — monitor `grafana-server` process RSS/VSZ growth correlated with inbound HTTP traffic spikes.

4. **HTTP 500 / OOM-related errors in Grafana logs** — repeated server errors or process restarts following large inbound requests.

5. **Single-source high-volume large-payload requests** — DoS via memory exhaustion typically manifests as a small number of source IPs sending very large bodies in rapid succession.

6. **Absence of `Content-Length` limiting or chunked transfer encoding abuse** — check WAF/proxy logs for requests without a declared Content-Length or with abnormally large declared sizes.
