### Recommended Observable Behaviour Patterns (Detection Proxies)

Because direct IoCs are unavailable, analysts should instrument for the following behavioural signals on NGINX Plus / NGINX Open Source instances:

1. **Unexpected worker process restarts** — The vulnerability enables denial-of-service via worker crash [1][2]. Monitoring `nginx` process supervisor logs for abnormal respawn events (especially correlated with slice-module-enabled vhosts) is the highest-confidence detection signal.

2. **Uninitialized memory disclosure in responses** — Responses from slice-enabled endpoints containing anomalous binary or non-printable content in unexpected positions may indicate triggered memory disclosure [1][2]. DLP or response inspection controls should be reviewed for these endpoints.

3. **High-volume requests to cached, slice-enabled endpoints** — Adversaries probing for the background cache update code path may generate elevated request rates to cacheable URIs. Correlate with `X-Slice-Range` or `Range` headers if applicable to your configuration.

4. **Configuration audit** — The prerequisite for exploitation is `ngx_http_slice_module` being enabled alongside unnamed regex captures [1][2]. Treat any confirmed matching configuration as a high-priority remediation target regardless of observed traffic.
