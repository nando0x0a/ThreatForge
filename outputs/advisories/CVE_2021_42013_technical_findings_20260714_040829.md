CVE-2021-42013 is a path traversal vulnerability in Apache HTTP Server that, under specific conditions, enables unauthenticated remote code execution. It represents a bypass of the incomplete fix applied in CVE-2021-41773, introduced in 2.4.49 and persisting in 2.4.50. Its CISA KEV listing and 9.8 CVSS score classify this as a Tier 1 critical priority requiring immediate action.

## Attack Vector

### Exploitation Mechanism

The vulnerability arises from insufficient normalisation of URL-encoded path sequences in Apache's path resolution logic. An attacker submits a crafted HTTP request containing double-encoded or otherwise obfuscated dot-dot sequences (`..`) that bypass `mod_alias` and `mod_rewrite` path restrictions, allowing traversal outside document root or alias-configured directories.

The incomplete patch for CVE-2021-41773 blocked single `%2e` encoding but failed to account for double-encoding (`%%32%65` → `%2e` → `.`), allowing the bypass.

### Exploitation Conditions

Two distinct exploitation paths exist:

**Path 1 — Directory Traversal (file disclosure):**
- Requires: A `mod_alias` configured directory (e.g., `Alias /icons /var/www/icons`) without an explicit `Require all denied` on the underlying filesystem path.
- Result: Arbitrary file read outside document root.

**Path 2 — Remote Code Execution:**
- Requires: `mod_cgi` or `mod_cgid` enabled AND a CGI-enabled directory accessible via `Alias` or `ScriptAlias` without `Require all denied`.
- Result: Unauthenticated OS command execution as the Apache process user (typically `www-data` or `apache`).

| **Source IP behaviour** | Rapid sequential requests, single IP testing multiple traversal depths (1–8 levels) |

### Endpoint / Process Telemetry

If CGI-based RCE is successful, expect the following process chain in endpoint telemetry:

httpd (apache2)
  └── sh / bash            ← spawned by mod_cgi executing shell via POST body
        ├── id             ← common first-stage recon command
        ├── wget / curl    ← payload retrieval
        ├── whoami
        └── <malware>      ← persistence mechanism or reverse shell

**Key process telemetry indicators:**
- Parent process: `httpd` or `apache2` (PID with network listening socket)
- Child process: `/bin/sh`, `/bin/bash`, `/usr/bin/python`, `/usr/bin/perl`
- Command-line arguments on child process containing: `wget`, `curl`, `nc`, `ncat`, `base64 -d`, `/dev/tcp/`
- File writes to world-writable directories: `/tmp/`, `/var/tmp/`, `/dev/shm/`
- Network connections from `apache2`/`httpd` process to external IPs (unusual — Apache should not initiate outbound connections)
